# 프로젝트 안내 (정보관리기술사 학습 노트)

순수 HTML/CSS/JS 정적 사이트(GitHub Pages). 빌드 도구 없음.

## 구조
- `index.html` (목록·검색), `topic.html` (상세, `?id=` 슬러그)
- `data/topics.json` — 주제 메타데이터(제목·카테고리·태그·요약·updated)
- `content/<id>.md` — 주제별 본문(마크다운, marked + DOMPurify로 렌더)
- `assets/js/main.js`(목록·검색·D-day), `assets/js/topic.js`(상세·이전/다음 네비)
- 렌더러: `assets/js/vendor/`(marked·DOMPurify self-host), mermaid·chart.js는 CDN 정확버전 고정

## 주제 작성 규칙 (중요)
**앞으로 추가·수정하는 모든 주제는 "논술형(답안형)"으로 작성한다.** 실제 시험 답안(약 2페이지)을
쓸 수 있는 깊이를 갖춰야 하며, 1교시 단답형 주제도 예외 없이 논술형으로 작성한다.

### 깊이 기준 (표 나열식 "요약 시트" 금지)
- **표만 나열하지 말 것.** 각 개념·항목은 표 앞뒤로 **산문(문단)으로 원리·이유·배경을 서술**한다.
  표는 비교·정리 보조 수단이며, "왜 그런지"는 문장으로 설명한다.
- **소항목(가/나/다/라)은 각각 문단 + (필요시)표**로 충분히 전개한다.
- **구체 사례·수치·예시**를 최소 1개 이상 포함한다.
- 비교는 항목 나열이 아니라 **차이가 생기는 이유**까지 설명한다.
- 결론(고려사항·시사점)은 **기술사 관점**(적용 전략·트레이드오프·전망·연계 기술)으로 심화한다.
- 목표 분량: **본문 100줄 이상**(단순 개념도 80줄 이상). 얇으면 보강한다.

### 구조
1. `# 제목`
2. `## 1. 개요` — 정의(> 인용구) + 등장 배경/필요성 **산문 서술** + (필요시) 특징
3. `## 2~4.` — 개념도(mermaid) + **각 요소를 문단으로 설명** + 표(보조) + 유형·절차·구성요소
   - 원 문제에 소문항(가/나/다/라)이 있으면 **모두, 깊이 있게 커버**
4. 비교 / 사례 (이유·맥락 서술 포함)
5. `## N. 고려사항 및 시사점` — 기술사 관점 심화
6. `---` 뒤 `> **한 줄 요약**: ...`

- 표(`|...|`)와 mermaid 다이어그램을 적극 활용하되 **산문 설명과 병행**(정의는 `>` 인용구로).
- 정확성 우선: 불확실하면 일반화하고 단정 표현을 피한다.

## 주제 추가 절차
1. `content/<id>.md` 작성(위 구조), `id`는 소문자 슬러그(`^[a-z0-9][a-z0-9-]*$`)
2. `data/topics.json`에 항목 추가(id·title·category·tags·summary·updated)
3. `id` 중복 금지, `content/<id>.md`와 1:1 정합 유지
4. 기출 주제는 태그에 회차(예: `133회`) 포함

## 검증
- JSON 유효성 + `content/<id>.md` 정합성 확인
- 로컬 서버(`python3 -m http.server`)로 렌더 확인(표·mermaid·이전/다음 네비)

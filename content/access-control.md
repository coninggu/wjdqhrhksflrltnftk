# 접근제어(Access Control)

## 1. 개요

### 가. 개념
> 인증된 주체(Subject)가 객체(Object)에 대해 **허가된 권한만큼만 접근**하도록 통제하는 정보보호 메커니즘. 정보보호는 크게 **암호화**와 **접근제어**로 분류.

### 나. 3요소
- **식별·인증 → 인가(Authorization) → 책임추적성(Accounting)**

## 2. 접근제어 정책

| 정책 | 내용 |
|---|---|
| **DAC**(임의적) | 소유자가 권한 부여, 유연하나 통제 약함 |
| **MAC**(강제적) | 보안등급·라벨 기반 강제(군·기밀) |
| **RBAC**(역할기반) | 역할에 권한 부여(기업 표준) |
| **ABAC**(속성기반) | 속성·상황(Context) 기반 동적 제어 |

## 3. 절차

```mermaid
flowchart LR
  S[주체] --> I[식별·인증]
  I --> A[인가<br/>정책 평가]
  A --> O[객체 접근 허용/거부]
  O --> L[로그·감사]
```

## 4. 구현 메커니즘

| 메커니즘 | 설명 |
|---|---|
| **ACL** | 객체별 접근 권한 목록 |
| **Capability** | 주체별 권한 토큰 |
| **보안 레이블** | 등급 비교(MAC) |
| **참조모니터** | 모든 접근 중재·강제(TCB) |

---

> **한 줄 요약**: 접근제어는 *식별·인증→인가→감사* 절차로 주체의 객체 접근을 통제하며, DAC·MAC·RBAC·ABAC 정책을 ACL·Capability·참조모니터로 구현한다.

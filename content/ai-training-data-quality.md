# AI 학습용 데이터셋 품질관리

## 1. 개요

### 가. 정의
> AI 학습용 데이터 허브 구축 과정에서 생성된 **학습용 데이터셋의 품질을 확보**하기 위한 활동으로, 데이터 수집·가공·검수 전 과정에서 정확성·일관성·충분성을 관리한다.

AI 성능은 결국 **데이터 품질에 수렴**한다(Garbage In, Garbage Out). 아무리 좋은 모델도 레이블이 틀리거나 편향된 데이터로 학습하면 신뢰할 수 없는 결과를 낸다. 특히 지도학습은 정답(레이블)의 품질이 곧 성능 상한이므로, 데이터셋 구축 단계의 체계적 품질관리가 모델 개발보다 선행·중요하다.

## 2. 주요 품질 확보 활동

```mermaid
flowchart LR
  A[수집] --> B[정제·가공] --> C[가공/라벨링] --> D[검수·검증] --> E[배포·활용]
  D -. 반려·재작업 .-> C
  style D fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 활동 | 내용 |
|---|---|
| **수집 관리** | 대표성·다양성 확보, 편향·저작권·개인정보 점검 |
| **정제·가공** | 결측·이상치 처리, 정규화, 중복 제거 |
| **라벨링(가공)** | 가이드라인 기반 어노테이션, 작업자 교육 |
| **검수·검증** | 교차검증, 작업자 간 일치도(IAA), 샘플링 검사 |
| **품질 지표** | 정확성·완전성·일관성·유효성 측정 |

## 3. 데이터 생애주기별 품질관리 절차

| 단계 | 품질관리 |
|---|---|
| **계획** | 품질 목표·기준·지표 정의 |
| **수집** | 원천 데이터 적합성·대표성 검증 |
| **가공·라벨링** | 표준 가이드라인 적용, 편향 점검 |
| **검수** | 다단계 검수(구축→검수→최종), 정량 기준 충족 |
| **활용·관리** | 버전 관리, 지속적 품질 모니터링·갱신 |

## 4. 시사점
- 라벨 품질이 곧 모델 성능 상한 — **검수·IAA가 핵심**
- 데이터 편향은 사회적 차별로 이어져 윤리·법적 리스크
- MLOps·데이터 중심 AI(Data-centric AI)로 지속 품질관리 자동화

---

> **한 줄 요약**: AI 학습용 데이터 품질관리는 *수집→가공·라벨링→검수→활용* 생애주기 단계마다 정확성·일관성·대표성을 관리하며, 라벨 검수·IAA·편향 점검이 모델 성능과 신뢰성의 관건이다.

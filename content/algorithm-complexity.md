# 알고리즘 복잡도와 O-Notation

## 1. 개요

### 가. 정의
> **알고리즘 복잡도**는 입력 크기(n)에 따른 자원 소모량으로, **시간 복잡도**(연산 횟수)와 **공간 복잡도**(메모리)로 구분한다. **O-Notation(빅오)** 은 입력이 커질 때의 **최악 증가율(점근 상한)** 을 표기한다.

### 나. 점근 표기법

| 표기 | 의미 |
|---|---|
| **O (빅오)** | 상한(worst case) — 가장 널리 사용 |
| **Ω (빅오메가)** | 하한(best case) |
| **Θ (빅세타)** | 상·하한 일치(average) |

| 유형 | 명칭 | 예시 | 평가 |
|---|---|---|---|
| **O(1)** | 상수 | 해시 조회, 배열 인덱스 | 최고 |

| **O(n²)** | 제곱 | 버블·삽입 정렬 | 나쁨 |
| **O(2ⁿ)** | 지수 | 부분집합, 재귀 피보나치 | 매우 나쁨 |
| **O(n!)** | 계승 | 외판원 완전탐색 | 최악 |

> 증가율: **O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)**

## 3. 케이스 분석

| 케이스 | 내용 |
|---|---|
| **최선(Best)** | 가장 빠른 경우(Ω) |
| **평균(Average)** | 기대 성능(Θ) |
| **최악(Worst)** | 보장 상한(O) — 설계 기준 |

## 4. 고려사항 및 시사점
- 대용량 데이터에선 **점근 복잡도가 실제 성능을 좌우**
- 시간·공간 **트레이드오프**(예: 캐싱은 공간↑ 시간↓)
- 상수항·저차항은 무시(점근적 관점), 실제는 상수·하드웨어도 고려
- P-NP 등 계산 복잡도 이론, 알고리즘 선택·최적화의 기준

---

> **한 줄 요약**: O-Notation은 입력 크기에 따른 알고리즘의 *최악 증가율(점근 상한)* 을 표기하며, *O(1)→O(log n)→O(n)→O(n log n)→O(n²)→O(2ⁿ)→O(n!)* 순으로 성능이 급격히 나빠진다.

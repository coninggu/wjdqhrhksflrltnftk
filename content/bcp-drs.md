# BCP 주요 지표와 DRS 구축 고려사항

## 1. 개요

### 가. 정의
- **BCP(Business Continuity Planning)**: 재해·장애 시 **핵심 업무의 연속성**을 보장하기 위한 계획·체계
- **DRS(Disaster Recovery System)**: BCP의 IT 실현 수단으로 시스템·데이터를 **복구**하는 재해복구 체계

### 나. 필요성
- 재해·사이버 공격 시 **업무 중단 최소화**, 규제(전자금융·재해경감) 대응

## 2. BCP 수립 절차와 주요 지표

```mermaid
flowchart LR
  B[BIA<br/>업무영향분석] --> R[위험평가]
  R --> S[복구전략 수립]
  S --> P[BCP 수립·훈련]
```

| 지표 | 의미 |
|---|---|
| **RTO**(목표복구시간) | 장애~복구까지 허용 시간 |
| **RPO**(목표복구시점) | 허용 가능한 데이터 손실 시점(백업 주기) |
| **RSO**(복구범위목표) | 복구 대상 업무·시스템 범위 |
| **MTD** | 최대허용중단시간(BIA 기반) |
| **RCO/RCapO** | 복구 통신·용량 목표 |

## 3. DRS 구축 유형(복구 수준)

| 유형 | RTO | 특징 |
|---|---|---|
| **Mirror Site** | 즉시(0) | 실시간 이중화(고비용) |
| **Hot Site** | 수시간 | 가동 상태 대기 |
| **Warm Site** | 수일 | 부분 구성 |
| **Cold Site** | 수주 | 공간·설비만(저비용) |

## 4. DRS 구축 핵심 고려사항

```mermaid
flowchart LR
  T[RTO/RPO 목표] --> D[센터 간 거리]
  D --> C[데이터 복제 방식]
  C --> E[전환·복구 절차]
  E --> M[모의훈련·검증]
```

| 고려사항 | 내용 |
|---|---|
| **RTO/RPO vs 비용** | 목표 수준과 투자 균형 |
| **센터 간 거리** | 동시 피해 회피(지역 분리) vs 동기복제 지연 |
| **데이터 복제** | 동기(무손실·지연↑)/비동기(성능↑·손실 가능) |
| **전환 절차** | 자동/수동 Failover·Failback, RunBook |
| **검증** | **정기 모의훈련**, 복구 가능성 실증 |

## 5. 시사점
- 클라우드 DR(파일럿 라이트·웜 스탠바이)로 **비용 효율적** 구축 확대
- BCP는 **문서화가 아닌 훈련·검증**으로 실효성 확보

---

> **한 줄 요약**: BCP는 *BIA 기반 RTO·RPO·RSO·MTD* 지표로 업무 연속성을 계획하고, DRS는 *복구 수준(Mirror~Cold)과 RTO/RPO·센터 거리·복제방식·모의훈련* 을 핵심 고려사항으로 구축한다.

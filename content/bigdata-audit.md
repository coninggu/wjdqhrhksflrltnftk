# 빅데이터 정보화 사업의 감리 점검항목

## 1. 개요

### 가. 배경
> 인공지능·빅데이터 등 지능정보 기술의 특성을 충분히 반영하지 못하는 기존 감리 기준의 한계를 보완하기 위해 지능정보기술 감리 실무 가이드가 발간되었다. 이 중 **빅데이터 정보화 사업의 분석·설계 단계** 에 대한 영역별 점검 항목을 다룬다.

빅데이터 사업의 감리가 일반 소프트웨어 감리와 다른 근본 지점은 '**데이터와 분석 모델이 산출물의 핵심**'이라는 데 있다. 전통적인 정보화 사업은 요구된 기능이 제대로 구현되었는지가 감리의 초점이지만, 빅데이터 사업은 어떤 데이터를 어떻게 확보·정제했고 그 위에서 어떤 분석 모델을 세웠는지가 성패를 좌우한다. 아무리 시스템이 잘 만들어져도 데이터가 대표성이 없거나 편향되고 분석 모델이 타당하지 않으면 결과를 신뢰할 수 없다. 그래서 기존의 기능 중심 점검만으로는 부족하고, **데이터의 품질·대표성과 분석 모델의 타당성** 을 중점적으로 점검하는 새로운 감리 기준이 필요했다.

### 나. 필요성
빅데이터·AI 사업이 공공 전반으로 확산되지만, 이를 검증할 감리 기준이 기능 중심에 머물러 있으면 데이터·분석의 결함을 놓친다. 신기술 특성을 반영한 점검 항목이 사업의 실질적 품질을 담보한다.

## 2. 분석·설계 단계 점검 개념

감리는 사업 단계별로 초점이 다르다. **분석 단계** 에서는 '무엇을, 어떤 데이터로 분석할 것인가'가 타당한지를 본다. 분석 요건과 목표가 명확한지, 활용할 데이터의 원천과 수집 대상이 적절하고 대표성·품질을 갖췄는지를 점검한다. **설계 단계** 에서는 '그것을 어떻게 구현할 것인가'가 적정한지를 본다. 데이터 저장·처리 아키텍처, 분석 알고리즘·모델 설계, 확장성, 보안·품질 방안을 점검한다.

## 3. 영역별 점검 항목

빅데이터 감리는 데이터·분석·아키텍처·보안·활용의 영역별로 분석·설계 단계를 점검한다. 특히 데이터 영역에서는 원천 데이터의 적정성과 대표성이, 분석 영역에서는 분석 요건과 모델 설계의 타당성이 핵심이다.

| 영역 | 분석 단계 점검 | 설계 단계 점검 |
|---|---|---|
| **데이터** | 데이터 원천·수집 대상 적정성, 품질·대표성 | 데이터 모델·저장 설계, 표준·품질 기준 |
| **분석(모델)** | 분석 요건·목표 정의 타당성 | 분석 알고리즘·모델 설계 적정성 |
| **아키텍처·인프라** | 처리 규모·성능 요건 | 빅데이터 플랫폼·확장성 설계 |
| **보안·품질** | 개인정보·비식별 요건 | 접근통제·품질관리 방안 |
| **활용** | 활용 목적·성과지표 | 시각화·서비스 연계 설계 |

## 4. 고려사항 및 시사점

1. **기능이 아닌 데이터·분석 모델 타당성 중심 점검**이 핵심이다. 데이터의 대표성·품질과 분석 모델의 적정성이 곧 결과의 신뢰성을 좌우하므로, 감리도 이 부분에 집중해야 한다.
2. **데이터 보안·윤리 점검이 강화**된다. 빅데이터는 개인정보를 다루는 경우가 많아, 비식별화·개인정보 보호 요건 충족 여부를 반드시 확인해야 한다.
3. **분석 결과의 신뢰성·재현성 확보**를 설계 단계에서 검증한다. 분석 모델이 재현 가능하고 편향 없이 신뢰할 만한 결과를 내는지를 사전에 점검해, 사업 종료 후의 실패를 예방한다.

---

> **한 줄 요약**: 빅데이터 정보화 사업 감리는 분석·설계 단계에서 *데이터(원천·품질·대표성)·분석모델(타당성)·아키텍처·보안·활용* 영역을 점검하며, 기능이 아닌 데이터와 분석 모델의 타당성·신뢰성 확인이 핵심이다.

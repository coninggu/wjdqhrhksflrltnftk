# 빅데이터 정보화 사업의 감리 점검항목

## 1. 개요

### 가. 배경
> 인공지능·빅데이터 등 지능정보 기술의 특성을 반영하지 못하는 기존 감리 기준의 한계를 보완하기 위해, 지능정보기술 감리 실무 가이드가 발간되었다. 이 중 **빅데이터 정보화 사업의 분석·설계 단계**에 대한 영역별 점검 항목을 다룬다.

빅데이터 사업 감리가 일반 SW 감리와 다른 점은 '**데이터가 산출물의 핵심**'이라는 데 있다. 일반 사업은 기능 구현이 중심이지만, 빅데이터 사업은 데이터의 수집·품질·분석 모델의 타당성이 성패를 좌우한다. 따라서 데이터·분석 관점의 점검이 강화되어야 한다.

## 3. 영역별 점검 항목

| 영역 | 분석 단계 점검 | 설계 단계 점검 |
|---|---|---|
| **데이터** | 데이터 원천·수집 대상 적정성, 품질·대표성 | 데이터 모델·저장 설계, 표준·품질 기준 |
| **분석(모델)** | 분석 요건·목표 정의 타당성 | 분석 알고리즘·모델 설계 적정성 |
| **아키텍처·인프라** | 처리 규모·성능 요건 | 빅데이터 플랫폼·확장성 설계 |
| **보안·품질** | 개인정보·비식별 요건 | 접근통제·품질관리 방안 |
| **활용** | 활용 목적·성과지표 | 시각화·서비스 연계 설계 |

## 4. 시사점
- 기능 중심이 아닌 **데이터·분석 모델 타당성** 중심 점검이 핵심
- 개인정보·비식별화 등 데이터 보안·윤리 점검 강화
- 분석 결과의 **신뢰성·재현성** 확보 여부를 설계 단계에서 확인

---

> **한 줄 요약**: 빅데이터 정보화 사업 감리는 분석·설계 단계에서 *데이터(원천·품질)·분석모델(타당성)·아키텍처·보안·활용* 영역을 점검하며, 데이터와 분석 모델의 타당성·신뢰성 확인이 핵심이다.

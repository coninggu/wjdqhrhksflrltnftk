# 빅데이터 플랫폼 아키텍처 설계

## 1. 개요

### 가. 정의
> **빅데이터 플랫폼 아키텍처**는 대용량·다양·고속의 데이터를 **수집→저장→처리→분석→시각화** 하는 전 과정을 지원하도록, 인프라·데이터 구조·입출력을 체계적으로 설계한 기술 구조다.

빅데이터 플랫폼 설계가 어려운 이유는 '**양(Volume)·다양성(Variety)·속도(Velocity)를 한 시스템이 감당해야 한다**'는 데 있다. 데이터는 테라·페타바이트로 커지고, 정형(DB)·반정형(로그)·비정형(영상·텍스트)이 뒤섞이며, 실시간 스트림과 대량 배치가 공존한다. 하나의 전통적 DBMS로는 이를 감당할 수 없다. 그래서 빅데이터 플랫폼은 계층적으로 설계된다. 확장 가능한 **인프라**(분산 클러스터·스토리지) 위에, 다양한 형태의 데이터를 담는 **데이터 구조**(데이터 레이크·웨어하우스)를 두고, 실시간·배치를 함께 처리하는 **입출력 구조**를 얹는다. 핵심 설계 원칙은 수평 확장성(스케일아웃), 데이터 특성별 저장·처리 분리, 그리고 배치와 실시간을 아우르는 처리 구조(람다·카파 아키텍처)다. 이 계층적·확장적 설계가 폭증하는 데이터를 안정적으로 다루는 열쇠다.

### 나. 설계 원칙
확장성(스케일아웃), 데이터 다양성 수용, 실시간·배치 처리 병행, 고가용성·내결함성, 데이터 거버넌스·보안이 핵심 원칙이다.

## 2. 인프라 구조 설계

인프라는 대량 데이터를 감당하기 위해 여러 노드로 이뤄진 **분산 클러스터**로 구성한다. 저장은 분산 파일 시스템(HDFS)·객체 스토리지, 처리는 병렬 분산 엔진(Hadoop·Spark)을 쓴다. 서버를 추가해 용량·성능을 늘리는 수평 확장이 기본이며, 노드 장애에도 견디는 복제·내결함성을 갖춘다. 최근에는 클라우드 관리형 서비스로 인프라 구축·운영 부담을 줄인다.

| 계층 | 기술 |
|---|---|
| **수집** | Kafka, Flume, NiFi |
| **저장** | HDFS, 객체 스토리지, 데이터 레이크 |
| **처리** | Hadoop(배치), Spark(인메모리) |
| **분석·서비스** | ML, BI, 시각화 |

## 3. 데이터 구조 분석과 입출력 구조 설계

**데이터 구조 분석**은 다룰 데이터의 형태를 분류하는 데서 시작한다. 정형(관계형)·반정형(JSON·로그)·비정형(영상·텍스트)을 구분하고, 원천·수집주기·품질을 파악한다. 원천 그대로 저장하는 **데이터 레이크**와, 정제·구조화해 분석하는 **데이터 웨어하우스**를 목적에 맞게 조합한다.

**입출력 구조 설계**는 데이터가 들어오고 나가는 방식을 정한다. 대량을 몰아 처리하는 **배치**와 실시간으로 흘려보내는 **스트리밍**을 함께 지원해야 한다. 이를 통합하는 대표 설계가 **람다 아키텍처**(배치 계층+속도 계층 병행)와 이를 단순화한 **카파 아키텍처**(스트림 중심)다.

| 구조 | 내용 |
|---|---|
| **데이터 레이크** | 원천 데이터 원형 저장(비정형 포함) |
| **데이터 웨어하우스** | 정제·구조화된 분석용 저장소 |
| **람다 아키텍처** | 배치+실시간 계층 병행 |
| **카파 아키텍처** | 스트림 중심 단일 처리 |

## 4. 고려사항 및 시사점

1. **확장성과 비용의 균형**이 관건이다. 데이터 증가에 대비한 스케일아웃 설계가 필수지만, 과도한 선투자는 낭비이므로 클라우드 탄력 확장(auto-scaling)으로 수요에 맞춰 조절한다.
2. **데이터 거버넌스·품질을 내재화**한다. 플랫폼이 아무리 좋아도 데이터 품질·보안·메타데이터 관리가 없으면 '데이터 늪(swamp)'이 되므로, 거버넌스를 설계 초기부터 반영한다. [[data-governance]]
3. **실시간·AI 요구가 커진다.** 실시간 분석·머신러닝 파이프라인(MLOps) 수요가 늘어, 스트리밍 처리와 데이터 레이크하우스(레이크+웨어하우스 통합)로 진화하고 있다.

---

> **한 줄 요약**: 빅데이터 플랫폼 아키텍처는 *수집→저장→처리→분석을 지원하는 확장적 계층 구조* 로, 분산 인프라·데이터 레이크/웨어하우스·배치와 실시간(람다/카파) 입출력을 설계하며, 확장성·거버넌스·실시간 대응이 핵심이다.

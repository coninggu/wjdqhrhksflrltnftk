# 클라우드 전환 사업의 단계별 감리

## 1. 개요

### 가. 정의
> 클라우드 전환(마이그레이션) 사업이 **목표·요구사항·보안·품질 기준에 부합**하는지 단계별로 점검·평가하여 위험을 통제하는 정보시스템 감리 활동.

### 나. 필요성
- 전환 과정의 **데이터 무결성·보안·성능·비용** 리스크 통제
- 공공·대형 사업의 품질·적정성 확보

## 2. 단계별 감리 방법과 검토 항목

```mermaid
flowchart LR
  P[계획/분석 감리] --> D[설계 감리] --> M[전환/이행 감리] --> O[운영/안정화 감리]
  style M fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 단계 | 감리 방법 | 주요 검토 항목 |
|---|---|---|
| **계획·분석** | 현황·요구 검토, 인터뷰 | 전환 대상 선정 적정성, 클라우드 적합성(6R), TCO·ROI, 보안요건 |
| **설계** | 산출물 검토, 아키텍처 리뷰 | 아키텍처·망분리, 데이터 이관 설계, 재해복구(DR), 표준 준수 |
| **전환·이행** | 이행 점검, 테스트 결과 검토 | 데이터 무결성·정합성, 성능, 롤백 계획, 보안 설정(CSAP) |
| **운영·안정화** | 운영 점검, 모니터링 검토 | 가용성·성능 SLA, 비용 최적화, 운영 이관, 보안 관제 |

## 3. 6R 전환 전략 (검토 기준)
| 전략 | 내용 |
|---|---|
| **Rehost** | 그대로 이전(Lift & Shift) |
| **Replatform** | 일부 최적화 후 이전 |
| **Refactor** | 클라우드 네이티브 재설계 |
| **Repurchase/Retire/Retain** | 재구매·폐기·유지 |

## 4. 시사점
- 감리는 **독립성·전문성** 기반 3자 관점 점검
- 데이터 이관 **무결성 검증**과 보안(CSAP)·비용 최적화가 핵심 리스크
- 전환 후 **운영 이관·거버넌스**까지 감리 범위 포함

---

> **한 줄 요약**: 클라우드 전환 감리는 *계획→설계→전환→운영* 단계별로 적합성·보안·데이터 무결성·성능·비용을 점검하여 전환 사업의 품질과 리스크를 통제한다.

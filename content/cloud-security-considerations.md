# 클라우드 서비스 도입 시 보안 고려 요소

## 1. 개요

### 가. 정의
> 기업이 클라우드 서비스를 도입할 때 **데이터·접근·규제·운영 전반에서 검토·확보해야 할 보안 요소**. 온프레미스와 다른 클라우드 특유의 위험에 대응한다.

클라우드 보안이 어려운 이유는 '**통제권의 일부가 클라우드 사업자(CSP)로 넘어가는**' 책임공유모델 때문이다. 인프라는 CSP가, 데이터·계정·설정은 이용자가 책임진다. 이 경계를 오해하면 "CSP가 알아서 하겠지"라는 사각지대가 생겨 대부분의 클라우드 사고(설정 오류·계정 탈취)가 발생한다.

## 3. 주요 보안 고려 요소

| 요소 | 고려 사항 |
|---|---|
| **데이터 보안** | 암호화(저장·전송), 데이터 위치·주권, 백업·파기 |
| **접근 통제(IAM)** | 최소권한, MFA, 계정·키 관리 |
| **설정 보안** | 잘못된 구성(공개 버킷 등) 방지, CSPM |
| **네트워크** | 망분리·보안그룹·방화벽, 제로트러스트 |
| **가시성·모니터링** | 로그·감사(CloudTrail), 위협 탐지 |
| **규제·인증** | CSAP·ISMS, 개인정보·산업 규제 준수 |
| **연속성** | 가용성·DR, Exit(종료·전환) 전략 |
| **공급망** | 컨테이너·이미지·IaC 보안, SBOM |

## 4. 시사점
- **책임공유모델 이해**가 클라우드 보안의 출발점
- 설정 오류·계정 관리가 실제 사고의 다수 → CSPM·IAM 강화
- CNAPP(클라우드 네이티브 보안)·제로트러스트로 통합 관리

---

> **한 줄 요약**: 클라우드 도입 시 *책임공유모델을 기반으로 데이터 암호화·IAM·설정보안·모니터링·규제준수·연속성* 을 검토해야 하며, 설정 오류·계정 관리가 최대 위험 요소다.

# 크리덴셜 스터핑(Credential Stuffing)

## 1. 개요

### 가. 정의
> 유출된 **계정·비밀번호 목록을 자동화 도구로 여러 사이트에 대입**해 로그인을 시도하는 공격. 사용자가 여러 서비스에 같은 비밀번호를 재사용하는 습관을 악용한다.

크리덴셜 스터핑이 위험한 이유는 '**정상 로그인처럼 보인다**'는 점이다. 무차별 대입(Brute Force)과 달리 이미 유효했던 실제 계정정보를 쓰므로 탐지가 어렵고, 비밀번호 재사용률이 높아 성공률이 상당하다. 한 사이트의 유출이 다른 모든 서비스로 번지는 연쇄 피해를 낳는다.

## 3. 유사 공격과 비교

| 구분 | 크리덴셜 스터핑 | 무차별 대입(Brute Force) |
|---|---|---|
| **입력** | 유출된 실제 계정목록 | 임의 조합 생성 |
| **성공률** | 높음(재사용 악용) | 낮음 |
| **탐지** | 어려움(정상 계정) | 상대적 용이(반복 실패) |

## 4. 대응 방안

| 대응 | 내용 |
|---|---|
| **다중인증(MFA)** | 비밀번호 유출돼도 2차 인증으로 차단(핵심) |
| **이상 탐지** | 비정상 로그인(대량·지역·속도) 탐지·차단 |
| **봇 방지** | CAPTCHA, 디바이스 핑거프린팅, Rate Limiting |
| **비밀번호 정책** | 유출 비밀번호 차단, 재사용 방지, 패스키 도입 |
| **모니터링** | 유출 계정 다크웹 모니터링·강제 재설정 |

## 5. 시사점
- **MFA·패스키(FIDO2)** 가 근본 방어 — 비밀번호 단일 의존 탈피
- 사용자 교육(비밀번호 재사용 금지)과 기술적 통제 병행
- 로그인 API에 대한 봇 트래픽 관리가 핵심

---

> **한 줄 요약**: 크리덴셜 스터핑은 *유출 계정을 자동 대입* 하는 공격으로 비밀번호 재사용을 악용해 탐지가 어려우며, **MFA·패스키·이상탐지·봇 방지** 로 대응한다.

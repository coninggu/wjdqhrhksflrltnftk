# 크리덴셜 스터핑(Credential Stuffing)

## 1. 개요

### 가. 정의
> 다른 사이트에서 **유출된 계정·비밀번호 목록을 자동화 도구로 여러 사이트에 대입**해 로그인을 시도하는 공격. 사용자가 여러 서비스에 같은 비밀번호를 재사용하는 습관을 정면으로 악용한다.

크리덴셜 스터핑이 특히 위험한 이유는 '**정상 로그인과 구분하기 어렵다**'는 데 있다. 무차별 대입(Brute Force)은 임의의 문자 조합을 찍어보므로 실패가 폭증해 탐지가 쉽지만, 크리덴셜 스터핑은 **한때 실제로 유효했던 계정정보**를 그대로 사용한다. 따라서 각 시도가 마치 진짜 사용자의 로그인처럼 보인다. 게다가 사람들의 비밀번호 재사용률이 매우 높아, 한 사이트에서 유출된 계정이 다른 사이트에서도 그대로 통하는 경우가 상당하다. 그 결과 한 곳의 유출이 사용자가 가입한 모든 서비스로 번지는 **연쇄 계정 탈취**를 일으킨다.

### 나. 등장 배경
대규모 개인정보 유출 사고가 반복되면서 수십억 건의 계정정보가 다크웹에 유통되었고, 이를 자동으로 대입하는 봇 도구가 상용화되면서 공격의 진입장벽이 크게 낮아졌다.

## 2. 공격 흐름

```mermaid
flowchart LR
  A["유출 계정목록 확보<br/>(다크웹)"] --> B["봇·자동화 대량 대입"]
  B --> C["성공 계정 선별<br/>(계정 탈취)"]
  C --> D["부정 이용·2차 판매"]
  style B fill:#fef3f2,stroke:#e11d48,stroke-width:2px
```

공격자는 유출 목록을 확보한 뒤, 프록시·봇넷으로 IP를 분산시켜가며 대량의 로그인을 자동 시도한다. 성공한 계정을 선별해 금전 탈취·포인트 사용 등으로 악용하거나 다시 판매한다. IP를 분산하는 이유는 한 IP에서 대량 시도 시 차단당하기 때문으로, 이 때문에 단순 IP 기반 차단만으로는 막기 어렵다.

## 3. 무차별 대입과의 비교

| 구분 | 크리덴셜 스터핑 | 무차별 대입(Brute Force) |
|---|---|---|
| **입력** | 유출된 실제 계정목록 | 임의 문자 조합 생성 |
| **성공률** | 높음(재사용 악용) | 낮음 |
| **탐지 난이도** | 어려움(정상 계정처럼 보임) | 상대적 용이(반복 실패) |
| **방어 초점** | 재사용 차단·MFA·봇 탐지 | 계정 잠금·복잡도 |

## 4. 대응 방안

크리덴셜 스터핑의 근본 방어는 '**비밀번호 하나에만 의존하지 않는 것**'이다. 비밀번호가 유출되어도 2차 인증이 있으면 로그인이 차단되므로 **다중인증(MFA)** 과 **패스키(FIDO2)** 가 가장 효과적이다. 여기에 비정상 로그인 패턴을 잡아내는 이상 탐지, 봇을 걸러내는 기술을 더한다.

| 대응 | 내용 |
|---|---|
| **다중인증(MFA)·패스키** | 비밀번호 유출돼도 2차 인증으로 차단(핵심) |
| **이상 탐지** | 대량·비정상 지역·비정상 속도 로그인 탐지·차단 |
| **봇 방지** | CAPTCHA, 디바이스 핑거프린팅, Rate Limiting |
| **비밀번호 정책** | 유출 비밀번호 차단(HIBP 연동), 재사용 방지 |
| **모니터링** | 다크웹 유출 계정 감시·강제 재설정 |

## 5. 고려사항 및 시사점

1. **MFA·패스키가 근본 대책**이다. 비밀번호라는 단일 인증 요소에 대한 의존에서 벗어나는 것이 크리덴셜 스터핑을 무력화하는 가장 확실한 길이다.
2. **사용자 습관과 기술 통제의 병행**이 필요하다. 비밀번호 재사용 금지 교육만으로는 한계가 있으므로, 서비스 측에서 유출 비밀번호를 차단하고 이상 로그인을 탐지해야 한다.
3. **로그인 API의 봇 트래픽 관리가 핵심**이다. IP 분산 공격에 대응해 디바이스·행위 기반 탐지, 위험 기반 인증(Risk-based Authentication)으로 의심스러운 로그인에만 추가 인증을 요구하는 방식이 확산되고 있다.

---

> **한 줄 요약**: 크리덴셜 스터핑은 *유출 계정을 자동 대입* 해 비밀번호 재사용을 악용하는 공격으로 정상 로그인처럼 보여 탐지가 어려우며, **MFA·패스키·이상탐지·봇 방지** 로 비밀번호 단일 의존에서 벗어나는 것이 근본 대응이다.

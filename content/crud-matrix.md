# CRUD 매트릭스(CRUD Matrix)

## 1. 개요

### 가. 정의
> 데이터모델의 **엔터티(데이터)** 와 프로세스(기능) 간의 **생성(Create)·조회(Read)·수정(Update)·삭제(Delete)** 관계를 행렬로 표현한 상관 분석 도구.

### 나. 사용 목적
- 데이터-프로세스 **정합성 검증**(누락·중복 식별)
- 트랜잭션 분석, 분산 설계, 접근권한·아카이빙 설계 근거 확보

## 2. 표현 방법

| 프로세스 \ 엔터티 | 회원 | 주문 | 상품 |
|---|---|---|---|
| **회원가입** | C | | |
| **상품조회** | | | R |
| **주문등록** | R | C | R |
| **주문취소** | | UD | |

- 각 셀에 해당 기능이 데이터에 수행하는 **C/R/U/D** 표기

## 3. 정합성 검증 규칙

```mermaid
flowchart LR
  C[각 엔터티 C 존재?] --> R[각 엔터티 R 존재?]
  R --> UD[U/D 프로세스 확인]
  UD --> F[고립 엔터티·빈 프로세스 점검]
```

| 점검 | 의미 |
|---|---|
| **각 엔터티에 C 존재** | 생성 프로세스 없으면 데이터 유입 불가 |
| **각 엔터티에 R 존재** | 활용되지 않는 데이터(불필요) 여부 |
| **U/D 존재 여부** | 상태변경·삭제 프로세스 확인 |
| **모든 행·열 최소 1개** | 고립 엔터티·빈 프로세스 방지 |

## 4. 활용 분야

| 분야 | 활용 |
|---|---|
| **업무·데이터 검증** | 요구-데이터 정합성 점검 |
| **트랜잭션 분석** | 트랜잭션 경계·동시성 설계 |
| **DB 분산 설계** | 프로세스별 접근 패턴으로 분할 |
| **접근권한 설계** | 역할별 CRUD 권한 매트릭스 기초 |

## 5. 고려사항 및 시사점
- 대규모 시스템은 **자동 생성 도구·CASE**로 매트릭스 관리
- 데이터 거버넌스·품질 관리의 기초 산출물
- MSA에서는 서비스-데이터 소유권(Bounded Context) 정의에도 응용

---

> **한 줄 요약**: CRUD 매트릭스는 *프로세스와 엔터티의 생성·조회·수정·삭제 관계를 행렬로 표현* 해 데이터-기능 정합성을 검증하고, 트랜잭션·분산·접근권한 설계의 근거로 활용되는 분석 도구다.

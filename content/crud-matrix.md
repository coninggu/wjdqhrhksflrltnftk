# CRUD 매트릭스(CRUD Matrix)

## 1. 개요

### 가. 정의
> 정보시스템의 **프로세스(기능)** 와 **엔터티(데이터)** 사이의 **생성(Create)·조회(Read)·수정(Update)·삭제(Delete)** 관계를 행렬로 교차 표현한 상관 분석 도구.

### 나. 등장 배경 및 사용 목적
정보공학(IE) 방법론에서 데이터 모델(무엇을 저장할까)과 프로세스 모델(무엇을 처리할까)은 서로 다른 시각에서 독립적으로 설계되기 때문에, 둘이 실제로 **맞물리는지**를 검증할 장치가 필요하다. 만약 어떤 엔터티에 값을 넣어 주는 프로세스가 하나도 없다면 그 데이터는 영원히 빈 채로 남고, 반대로 어떤 프로세스가 존재하지 않는 데이터를 참조하도록 설계됐다면 실행이 불가능하다. CRUD 매트릭스는 이런 **데이터-기능 간 정합성의 구멍**을 행렬이라는 한눈에 보이는 형태로 드러내, 누락·중복·고립을 조기에 식별한다. 나아가 트랜잭션 경계 설정, DB 분산 설계, 접근권한 설계, 아카이빙 정책의 근거 자료로 재활용된다.

## 2. 표현 방법

행에는 프로세스, 열에는 엔터티를 놓고, 각 교차 셀에 그 프로세스가 해당 데이터에 수행하는 연산을 C/R/U/D로 표기한다. 하나의 프로세스가 여러 연산을 하면 함께 적는다(예: 주문취소는 주문을 수정·삭제하므로 UD). 아래는 간단한 쇼핑몰 예시로, "주문등록"은 회원·상품을 조회(R)하고 주문을 생성(C)하는 복합 트랜잭션임이 한 행에서 드러난다.

| 프로세스 \ 엔터티 | 회원 | 주문 | 상품 |
|---|---|---|---|
| **회원가입** | C | | |
| **상품조회** | | | R |
| **주문등록** | R | C | R |
| **주문취소** | | UD | |

## 3. 정합성 검증 규칙

```mermaid
flowchart LR
  C[각 엔터티 C 존재?] --> R[각 엔터티 R 존재?]
  R --> UD[U/D 프로세스 확인]
  UD --> F[고립 엔터티·빈 프로세스 점검]
```

검증의 핵심은 "모든 엔터티가 생애주기 전체를 갖는가"와 "모든 프로세스가 실제 데이터를 다루는가"를 대칭적으로 확인하는 것이다. **각 엔터티 열에 C가 하나는 있어야** 한다. 생성 프로세스가 없는 엔터티는 데이터가 유입될 경로가 없어 설계 누락을 뜻한다. **각 엔터티 열에 R이 하나는 있어야** 한다. 아무도 조회하지 않는 데이터는 저장할 이유가 없는 불필요 데이터일 수 있으므로 재검토 대상이다. **U/D의 유무**는 상태 변경이나 삭제가 업무적으로 필요한지를 점검한다. 마지막으로 **모든 행·열이 최소 하나의 표기**를 가져야 하는데, 표기가 전혀 없는 열은 어떤 기능도 쓰지 않는 **고립 엔터티**, 표기가 전혀 없는 행은 데이터를 건드리지 않는 **빈(허수) 프로세스**로 오류 후보다. 위 예시에서 상품 열에 C가 없다면 "상품등록" 프로세스 누락을 발견하는 식이다.

| 점검 규칙 | 의미 | 위반 시 |
|---|---|---|
| **각 엔터티에 C 존재** | 데이터 유입 경로 확보 | 생성 프로세스 누락 |
| **각 엔터티에 R 존재** | 데이터 활용 여부 확인 | 불필요 데이터 의심 |
| **U/D 존재 여부** | 상태변경·삭제 필요성 | 관리 프로세스 검토 |
| **모든 행·열 최소 1개** | 고립·허수 방지 | 고립 엔터티·빈 프로세스 |

## 4. 활용 분야

CRUD 매트릭스는 정합성 검증에 그치지 않고 후속 설계의 입력으로 넓게 쓰인다. 프로세스별 CRUD 패턴을 분석하면 하나의 논리적 작업 단위(트랜잭션) 경계를 도출할 수 있고, 여러 프로세스가 같은 데이터에 동시에 U를 수행하면 **동시성 제어(락)** 설계의 근거가 된다. 특정 엔터티군을 특정 프로세스군만 접근한다면 그 경계를 따라 **DB를 분산·분할**하면 트래픽 지역성이 높아진다. 역할별로 어떤 데이터에 어떤 CRUD 권한을 줄지도 이 매트릭스에서 출발한다.

| 분야 | 활용 방식 |
|---|---|
| **업무·데이터 검증** | 요구사항-데이터 정합성 점검 |
| **트랜잭션 분석** | 트랜잭션 경계·동시성 설계 |
| **DB 분산 설계** | 접근 패턴 기반 분할·배치 |
| **접근권한 설계** | 역할별 CRUD 권한 매트릭스 기초 |

## 5. 고려사항 및 시사점
- **자동화·도구화**: 엔터티·프로세스가 수백 개인 대규모 시스템에서는 수작업 관리가 불가능하므로 CASE 도구나 리포지토리 기반 자동 생성을 활용해 모델 변경과 매트릭스를 동기화한다.
- **데이터 거버넌스의 기초 산출물**: 데이터 소유권·품질·수명주기 관리를 위한 출발점으로, 데이터 계보(lineage) 관리와 연결된다.
- **MSA로의 응용**: 마이크로서비스 아키텍처에서는 CRUD 패턴 분석이 서비스가 소유해야 할 데이터 경계(**Bounded Context**)를 정의하는 데 응용되어, "데이터를 소유한 서비스만 그 데이터를 쓴다"는 원칙의 설계 근거가 된다.

---

> **한 줄 요약**: CRUD 매트릭스는 *프로세스와 엔터티의 생성·조회·수정·삭제 관계를 행렬로 교차 표현* 해 데이터-기능 정합성(누락·고립·허수)을 검증하고, 트랜잭션·분산·접근권한 설계와 MSA의 Bounded Context 정의까지 확장되는 상관 분석 도구다.

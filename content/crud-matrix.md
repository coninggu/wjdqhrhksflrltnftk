# CRUD 매트릭스(CRUD Matrix)

## 1. 개요

### 가. 정의
> 데이터모델의 **엔터티(데이터)** 와 프로세스(기능) 간의 **생성(Create)·조회(Read)·수정(Update)·삭제(Delete)** 관계를 행렬로 표현한 상관 분석 도구.

### 나. 사용 목적
- 데이터-프로세스 **정합성 검증**, 누락·중복 기능 식별, 분산 설계·권한 설계 근거 확보

## 2. 표현 방법

| 프로세스 \ 엔터티 | 회원 | 주문 | 상품 |
|---|---|---|---|
| **회원가입** | C | | |
| **상품조회** | | | R |
| **주문등록** | R | C | R |
| **주문취소** | | UD | |

- 셀에 해당 기능이 데이터에 수행하는 **C/R/U/D** 표기

## 3. 검증(정합성) 규칙

| 점검 | 의미 |
|---|---|
| **각 엔터티에 C 존재** | 생성 프로세스 없으면 데이터 유입 불가 |
| **각 엔터티에 R 존재** | 활용되지 않는 데이터 여부 |
| **U/D 존재 여부** | 상태변경·삭제 프로세스 확인 |
| **모든 열·행 최소 1개** | 고립 엔터티·빈 프로세스 방지 |

## 4. 활용
- **업무·데이터 검증**, 트랜잭션 분석, DB 분산·아카이빙 설계, 접근권한 매트릭스 기초

---

> **한 줄 요약**: CRUD 매트릭스는 *프로세스와 엔터티의 생성·조회·수정·삭제 관계를 행렬로 표현* 해 데이터-기능 정합성을 검증하는 분석 도구다.

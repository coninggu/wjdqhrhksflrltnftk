# 데이터 통합 및 마이그레이션 — 무결성·정합성 확보

## 1. 개요

### 가. 배경
> 시스템 교체·통합 시 데이터를 옮기는 **데이터 마이그레이션**에서, 데이터가 유실·왜곡 없이 정확히 이관되도록 **무결성과 정합성을 확보**하는 것이 성공의 핵심이다. 데이터가 곧 자산인 만큼, 이관 오류는 서비스 신뢰를 직접 훼손한다.

데이터 마이그레이션이 어려운 근본 이유는 '**한 번 잘못 옮기면 되돌리기 어렵고, 그 오류가 계속 서비스에 영향을 미친다**'는 데 있다. 수년간 쌓인 데이터를 새 시스템 구조로 옮길 때는, 원본과 대상의 데이터 형식·규칙이 달라 값이 잘리거나 왜곡되고, 일부가 누락되며, 서로 연결된 데이터의 관계가 깨지기 쉽다. 예를 들어 주문 데이터는 옮겼는데 연결된 고객 데이터가 누락되면 참조 무결성이 깨진다. 그래서 마이그레이션은 단순히 데이터를 복사하는 것이 아니라, 이관 전후로 **데이터가 정확하고 완전하며 서로 일관됨을 검증** 하는 체계적 과정이 되어야 한다. 이 검증 없이 오픈하면 대규모 시스템 장애의 주요 원인이 된다.

### 나. 무결성과 정합성의 차이
두 개념은 자주 혼동되지만 다르다. **무결성(Integrity)** 은 데이터가 정확하고 유효하며 규칙(제약조건)을 만족하는 상태이고, **정합성(Consistency)** 은 여러 곳에 있는 데이터가 서로 모순 없이 일치하는 상태다.

| 구분 | 무결성(Integrity) | 정합성(Consistency) |
|---|---|---|
| **초점** | 데이터 자체의 정확·유효성 | 데이터 간 일치·모순 없음 |
| **예** | 나이가 음수 아님, 필수값 존재 | 원본과 대상 건수·값 일치 |
| **위반 시** | 잘못된 값 | 데이터 불일치·모순 |

## 3. 데이터 값 진단 프로파일링의 중점 분석 관점

이관 전 원본 데이터의 실제 상태를 진단하는 **데이터 프로파일링** 은 여러 관점에서 값을 분석한다.

| 관점 | 내용 |
|---|---|
| **값 분포·범위** | 이상치·범위 초과 값 파악 |
| **결측·널** | 누락·빈 값 비율 |
| **유일성·중복** | 키 중복, 유일성 위반 |
| **형식·패턴** | 데이터 형식(날짜·코드) 일관성 |
| **참조 무결성** | 연결된 데이터의 관계 유효성 |

프로파일링으로 원본의 문제(더러운 데이터)를 미리 파악해야, 이관 시 정제·변환 계획을 제대로 세울 수 있다.

## 4. 마이그레이션 검증 테스트 방법

| 방법 | 내용 |
|---|---|
| **건수 검증** | 원본-대상 레코드 수 일치 확인 |
| **값 비교** | 샘플·전수 값 대조(체크섬·해시) |
| **집계 검증** | 합계·평균 등 집계값 일치 |
| **참조 무결성** | 관계·외래키 유효성 검증 |
| **업무 검증** | 실제 업무 시나리오로 결과 확인 |

## 5. 고려사항 및 시사점

1. **검증이 마이그레이션의 성패를 좌우**한다. 이관 자체보다 이관 후 무결성·정합성 검증이 핵심이며, 건수·값·집계·참조 무결성을 다층으로 검증해야 오류를 놓치지 않는다.
2. **리허설과 롤백 계획이 필수**다. 실제 전환 전에 리허설로 절차·시간·오류를 점검하고, 문제 발생 시 되돌릴 롤백 계획을 갖춰 위험을 통제한다.
3. **원본 데이터 정제가 선행**되어야 한다. 프로파일링으로 발견한 더러운 데이터를 정제하지 않고 옮기면 문제가 그대로 이전되므로, 이관은 데이터 품질을 높이는 기회이기도 하다.

---

> **한 줄 요약**: 데이터 마이그레이션은 *무결성(값의 정확성)과 정합성(데이터 간 일치)* 을 확보하는 것이 핵심으로, 프로파일링으로 원본을 진단하고 건수·값·집계·참조 무결성을 다층 검증하며 리허설·롤백으로 안전하게 전환한다.

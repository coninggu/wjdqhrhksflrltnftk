# RDBMS 데이터 모델링

## 1. 개요

### 가. 개념·단계별 수행내용
> 현실 업무를 데이터 구조로 추상화해 **일관·무결한 DB 구조**를 설계하는 과정.

```mermaid
flowchart LR
  C[개념적 모델링<br/>ERD·엔터티] --> L[논리적 모델링<br/>정규화·관계]
  L --> P[물리적 모델링<br/>테이블·인덱스]
```

| 단계 | 수행 내용 |
|---|---|
| **개념적** | 핵심 엔터티·관계 식별(ERD), 업무 중심 |
| **논리적** | 속성·키 정의, **정규화**, 관계 설정 |
| **물리적** | 테이블·컬럼·인덱스·파티션 등 DBMS 반영 |

## 2. 식별(Identifying) vs 비식별(Non-Identifying) 관계

| 구분 | 식별 관계 | 비식별 관계 |
|---|---|---|
| **개념** | 부모 PK가 자식 **PK의 일부** | 부모 PK가 자식 **일반 속성(FK)** |
| **표기** | 실선 | 점선 |
| **종속성** | 강한 종속(부모 없이 존재 불가) | 약한 종속(독립 존재 가능) |
| **예** | 주문-주문상세 | 부서-사원 |

## 3. 모델링 시 고려사항
- **정규화 vs 반정규화**(성능 균형), 적절한 **키·인덱스** 설계
- **데이터 무결성**(개체·참조·도메인), 이력·표준화, 확장성

## 4. 시사점
- 정규화로 무결성 확보 후, 성능 요구에 따라 선택적 반정규화 적용

---

> **한 줄 요약**: 데이터 모델링은 *개념→논리→물리* 단계로 진행되며, 식별관계는 부모 PK가 자식 PK에 포함(강한 종속)·비식별은 일반 FK(약한 종속)로 구분하고, 정규화와 무결성·성능을 균형 있게 고려한다.

# 데이터 시각화(Data Visualization)

## 1. 개요

### 가. 정의
> 데이터를 **차트·그래프·지도 등 시각적 형태로 표현**하여 패턴·추세·관계를 직관적으로 인식하고 통찰(Insight)을 도출하도록 돕는 기법.

시각화의 힘은 인간이 숫자의 나열보다 **시각적 패턴을 훨씬 빠르게 인지**한다는 데 있다. 수백 행의 표에서는 보이지 않던 추세·이상치·상관관계가 하나의 그래프로 즉시 드러난다. 좋은 시각화는 '보기 좋은 그림'이 아니라 '**데이터의 의미를 정확하고 효율적으로 전달**'하는 커뮤니케이션 도구다.

## 2. 원리 및 절차

```mermaid
flowchart LR
  A[데이터 수집·정제] --> B[분석 목적 정의] --> C[시각 표현 매핑] --> D[차트 선택·제작] --> E[해석·공유]
  style C fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 원리 | 내용 |
|---|---|
| **정확성** | 데이터를 왜곡 없이 표현(축 조작 금지) |
| **명확성** | 전달 메시지가 분명하게 |
| **효율성** | 최소 잉크로 최대 정보(데이터-잉크 비율) |
| **심미성** | 가독성 높은 색·레이아웃 |

## 3. 유형

| 목적 | 유형 | 예 |
|---|---|---|
| **비교** | 막대·방사형 차트 | 카테고리별 매출 |
| **추세(시간)** | 선·영역 차트 | 시계열 변화 |
| **구성(비율)** | 파이·도넛·트리맵 | 점유율 |
| **관계** | 산점도·버블 | 상관관계 |
| **분포** | 히스토그램·박스플롯 | 데이터 분포 |
| **공간** | 지도·히트맵 | 지역별 데이터 |

## 4. 효과적인 시각화를 위한 효율화 방안
- 목적·대상(청중)에 맞는 **차트 유형 선택**(데이터 특성 우선)
- 불필요한 요소(차트정크) 제거, 색·레이블 최소·일관 사용
- 인터랙티브·대시보드(드릴다운)로 탐색성 강화
- 접근성(색맹 고려 팔레트)·모바일 반응형 고려

## 5. 시사점
- BI 도구(Tableau·Power BI)와 데이터 스토리텔링으로 의사결정 지원
- 왜곡·과장(오해 유발) 방지를 위한 시각화 윤리 중요

---

> **한 줄 요약**: 데이터 시각화는 *정확·명확·효율·심미* 원리에 따라 목적별 차트(비교·추세·구성·관계·분포·공간)를 선택해 데이터의 통찰을 직관적으로 전달하는 커뮤니케이션 기법이다.

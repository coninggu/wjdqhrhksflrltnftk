# DSML 프로젝트와 MLOps

## 1. 개요

### 가. 정의
> **DSML(Data Science & Machine Learning)** 은 데이터 사이언스와 기계학습을 융합해 데이터 기반 문제를 해결하는 프로젝트이며, **MLOps** 는 ML 모델의 개발-배포-운영을 자동화·표준화하는 방법론이다.

DSML 프로젝트가 어려운 이유는 '**실험은 성공해도 운영에서 실패**'하기 때문이다. 노트북에서 잘 되던 모델이 실제 서비스에서 데이터 변화·재학습·모니터링 부재로 무너진다. MLOps는 이 '연구와 운영의 간극'을 메워, 모델을 지속 가능한 제품으로 만든다.

## 2. DSML 프로젝트 수명주기 (1)

```mermaid
flowchart LR
  B[비즈니스 이해] --> D[데이터 수집·이해] --> P[데이터 준비] --> M[모델링] --> E[평가] --> De[배포·운영]
  De -. 모니터링·재학습 .-> B
  style M fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 단계 | 내용 |
|---|---|
| **비즈니스 이해** | 문제 정의, 목표·성공기준 |
| **데이터 수집·이해** | 데이터 확보·탐색(EDA) |
| **데이터 준비** | 정제·특징공학·분할 |
| **모델링** | 알고리즘 선택·학습·튜닝 |
| **평가** | 성능·비즈니스 지표 검증 |
| **배포·운영** | 서빙, 모니터링, 재학습 |

> CRISP-DM을 기반으로 하되 배포·운영(MLOps)이 강화된 순환 구조다.

## 3. MLOps (2)
> DevOps를 ML에 확장한 것으로, **데이터·모델·코드의 3요소를 버전관리하고 학습-배포-모니터링을 자동화**한다.

| 구성 | 내용 |
|---|---|
| **CI/CD/CT** | 지속통합·배포 + **지속학습(Continuous Training)** |
| **파이프라인** | 데이터→학습→평가→배포 자동화 |
| **모델 레지스트리** | 모델 버전·메타데이터 관리 |
| **모니터링** | 성능·데이터 드리프트·편향 감시 |
| **거버넌스** | 재현성·감사·책임 추적 |

## 4. 시사점
- **데이터 드리프트 감지·재학습**이 운영 성패의 핵심
- 데이터·모델·코드의 재현성(reproducibility) 확보
- MLOps 성숙도(수동→자동 파이프라인→완전 자동)로 단계적 고도화

---

> **한 줄 요약**: DSML 프로젝트는 *비즈니스 이해→데이터→모델링→배포·운영* 의 순환 수명주기를 거치며, MLOps로 데이터·모델·코드를 버전관리하고 학습-배포-모니터링-재학습을 자동화해 모델을 지속 가능한 제품으로 만든다.

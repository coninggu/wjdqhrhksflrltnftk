# DSML 프로젝트와 MLOps

## 1. 개요

### 가. 정의
> **DSML(Data Science & Machine Learning)** 은 데이터 사이언스와 기계학습을 융합해 데이터 기반 문제를 해결하는 프로젝트이며, **MLOps** 는 ML 모델의 개발-배포-운영을 자동화·표준화하는 방법론이다.

DSML 프로젝트가 유독 어려운 이유는 '**실험은 성공해도 운영에서 실패한다**'는 데 있다. 데이터 과학자가 노트북에서 훌륭한 정확도의 모델을 만들어도, 그것을 실제 서비스에 올려 지속적으로 가치를 내게 하는 것은 전혀 다른 문제다. 실제 데이터는 시간이 지나며 학습 시점과 달라지고(드리프트), 모델은 주기적으로 재학습·재배포되어야 하며, 성능을 계속 감시해야 한다. 이 '**연구와 운영 사이의 간극(Research-Production Gap)**'을 메우는 것이 MLOps다. 소프트웨어 개발에서 DevOps가 코드 배포를 자동화했듯, MLOps는 데이터·모델·코드라는 세 요소를 관리해 모델을 지속 가능한 제품으로 만든다.

### 나. 필요성
데이터·AI 기반 의사결정이 확산되면서, 일회성 모델이 아니라 지속적으로 신뢰할 수 있는 예측을 제공하는 시스템이 필요하다. MLOps 없이는 모델이 배포 후 방치되어 성능이 서서히 저하되고 결국 잘못된 판단을 낳는다.

## 2. DSML 프로젝트 수명주기

```mermaid
  De -. 모니터링·재학습 .-> B
  style M fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

DSML 프로젝트는 CRISP-DM을 기반으로 하되 배포·운영이 강화된 순환 구조를 따른다. 먼저 풀려는 문제와 성공 기준을 정의하고(비즈니스 이해), 데이터를 확보·탐색(EDA)한 뒤, 정제·특징공학으로 준비하고, 모델을 학습·튜닝한다. 성능과 비즈니스 가치를 평가한 후 배포·운영하며, 운영 중 성능을 모니터링해 필요하면 다시 처음으로 돌아가 재학습한다. 중요한 것은 이것이 한 번 끝나는 직선이 아니라 **계속 도는 순환** 이라는 점이다.

| 단계 | 내용 |
|---|---|
| **비즈니스 이해** | 문제 정의, 목표·성공기준 |
| **데이터 수집·이해** | 데이터 확보·탐색(EDA) |
| **데이터 준비** | 정제·특징공학·분할 |
| **모델링** | 알고리즘 선택·학습·튜닝 |
| **평가** | 성능·비즈니스 지표 검증 |
| **배포·운영** | 서빙, 모니터링, 재학습 |

## 3. MLOps(기계학습 운영화)

> DevOps를 ML로 확장한 것으로, **데이터·모델·코드 세 요소를 버전관리하고 학습-배포-모니터링을 자동화**한다. 핵심은 지속적 통합·배포에 더해 **지속적 학습(Continuous Training)** 이 추가된다는 점이다.

일반 소프트웨어는 코드만 관리하면 되지만, ML은 코드가 같아도 데이터가 바뀌면 결과가 달라지므로 데이터·모델까지 함께 버전관리해야 한다. MLOps는 데이터 파이프라인→학습→평가→배포를 자동화하고, 모델 레지스트리로 버전을 관리하며, 운영 중 성능·데이터 드리프트·편향을 지속 감시해 필요시 자동 재학습한다.

| 구성 | 내용 |
|---|---|
| **CI/CD/CT** | 지속통합·배포 + 지속학습(Continuous Training) |
| **파이프라인** | 데이터→학습→평가→배포 자동화 |
| **모델 레지스트리** | 모델 버전·메타데이터 관리 |
| **모니터링** | 성능·데이터 드리프트·편향 감시 |
| **거버넌스** | 재현성·감사·책임 추적 |

## 4. 고려사항 및 시사점

1. **데이터 드리프트 감지·재학습이 운영 성패를 좌우**한다. 실제 데이터가 학습 시점과 달라지면 성능이 조용히 무너지므로, 이를 감지해 재학습하는 자동화가 MLOps의 핵심이다.
2. **재현성(Reproducibility) 확보**가 신뢰의 기반이다. 데이터·모델·코드를 버전관리해 "그때 그 결과가 왜 나왔는지"를 재현·감사할 수 있어야 한다.
3. **MLOps 성숙도로 단계적 고도화**한다. 수동 프로세스에서 부분 자동화를 거쳐 완전 자동 파이프라인으로 발전하며, 조직 역량에 맞춰 점진적으로 수준을 높인다.

---

> **한 줄 요약**: DSML 프로젝트는 *비즈니스 이해→데이터→모델링→배포·운영* 의 순환 수명주기를 거치며, MLOps로 데이터·모델·코드를 버전관리하고 학습-배포-모니터링-재학습을 자동화(CT)해 모델을 지속 가능한 제품으로 만드는 것이 핵심이다.

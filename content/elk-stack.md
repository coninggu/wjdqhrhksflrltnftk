# ELK 스택 (Elasticsearch · Logstash · Kibana)

## 1. 개요

### 가. 정의
> 대용량 **로그·데이터의 수집·저장·검색·시각화**를 제공하는 오픈소스 스택. Beats를 포함해 **Elastic Stack**으로 불린다.

### 나. 필요성
- 분산 시스템·MSA의 **로그 통합·실시간 분석·관측성** 요구

## 2. 구성 요소

```mermaid
flowchart LR
  B[Beats<br/>수집 에이전트] --> L[Logstash<br/>수집·변환]
  L --> E[Elasticsearch<br/>색인·검색]
  E --> K[Kibana<br/>시각화·대시보드]
```

| 구성 | 역할 |
|---|---|
| **Beats** | 경량 수집기(Filebeat·Metricbeat) |
| **Logstash** | 데이터 수집·파싱·변환(Filter)·적재 |
| **Elasticsearch** | 분산 검색엔진(역색인, 실시간 검색·집계) |
| **Kibana** | 검색·시각화·대시보드·알림 |

## 3. 데이터 흐름·핵심 기술

| 기술 | 설명 |
|---|---|
| **역색인(Inverted Index)** | 빠른 전문 검색 |
| **분산·샤딩** | 수평 확장·고가용성(복제) |
| **파이프라인** | Grok·필터로 비정형 로그 구조화 |

## 4. 활용 분야

| 분야 | 활용 |
|---|---|
| **로그 분석** | 애플리케이션·시스템 로그 통합 |
| **보안(SIEM)** | 이상 탐지·위협 분석 |
| **APM·관측성** | 성능 모니터링·트레이싱 |

## 5. 고려사항 및 시사점
- 대용량 시 **리소스·인덱스 수명주기(ILM)** 관리, 저장비용
- 라이선스 변경(SSPL) → **OpenSearch** 대체 검토
- OpenTelemetry·관측성 표준과 연계

---

> **한 줄 요약**: ELK 스택은 *Beats/Logstash(수집·변환)→Elasticsearch(색인·검색)→Kibana(시각화)* 로 대용량 로그를 실시간 검색·분석하는 오픈소스 스택으로, 로그 분석·SIEM·관측성에 활용된다.

# 데이터 저장: 파일 · 데이터베이스 · 블록체인 비교

## 1. 개요

### 가. 정의
> 데이터를 저장·관리하는 세 가지 대표 방식으로, **파일**은 운영체제 파일 단위 저장, **데이터베이스(DBMS)** 는 구조화된 데이터의 통합 관리, **블록체인**은 분산 노드에 체인으로 연결된 불변 원장을 저장한다.

세 방식은 '**신뢰를 어디에 두는가**'가 다르다. 파일은 애플리케이션이, DBMS는 중앙 서버(관리자)가 데이터를 통제한다. 블록체인은 중앙 관리자 없이 **분산 합의**로 신뢰를 만든다. 이 차이가 무결성·성능·가용성의 특성을 결정한다.

## 2. 비교

| 구분 | 파일 | 데이터베이스 | 블록체인 |
|---|---|---|---|
| **구조** | 파일 단위 | 스키마·관계형 | 블록 체인(분산 원장) |
| **관리 주체** | 애플리케이션 | 중앙 DBMS | 분산 노드(탈중앙) |
| **무결성** | 낮음(중복·불일치) | 트랜잭션(ACID) | **불변성·위변조 방지** |
| **동시성·통합** | 취약 | 강함(병행제어) | 합의 지연 |
| **성능** | 단순·빠름 | 최적화(인덱스) | 느림(합의 오버헤드) |
| **가용성** | 단일 장애 | 이중화 | **탈중앙·고가용** |
| **적합** | 단순·소량 | 정형·통합 업무 | 신뢰 필요·탈중앙(거래·이력) |

## 3. 시사점
- **무결성·투명성**이 핵심이면 블록체인, **성능·통합**이면 DBMS, **단순 저장**은 파일
- 블록체인은 위변조 방지·투명성에 강하나 성능·저장 비용이 한계
- 하이브리드(온체인 해시 + 오프체인 데이터)로 절충 활용

---

> **한 줄 요약**: 파일(앱 관리·단순), DBMS(중앙 통합·ACID), 블록체인(분산 원장·불변성)은 신뢰 주체와 무결성·성능이 다르며, 통합·성능은 DBMS, 위변조 방지·탈중앙 신뢰는 블록체인이 적합하다.

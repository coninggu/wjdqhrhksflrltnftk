# FIPS 140-2 (암호모듈 보안 요구사항)

## 1. 개요

### 가. 정의
> 미국 NIST의 **암호모듈(Cryptographic Module) 보안 요구사항** 표준. 하드웨어·소프트웨어 암호모듈의 보안 수준을 4단계로 검증(CMVP).

### 나. 필요성
- 정부·금융의 암호 제품 **신뢰성 검증**, 조달 요건

## 2. 레벨 분류(Level 1~4)

```mermaid
flowchart LR
  L1[Level 1<br/>기본 알고리즘] --> L2[Level 2<br/>변조 증거·역할인증]
  L2 --> L3[Level 3<br/>변조 대응·물리보호]
  L3 --> L4[Level 4<br/>완전 봉인·환경공격 대응]
```

| 레벨 | 요구사항 |
|---|---|
| **Level 1** | 승인된 알고리즘 사용(물리보호 없음) |
| **Level 2** | 변조 증거(Tamper-evidence), 역할 기반 인증 |
| **Level 3** | 변조 대응(키 삭제), 물리적 보호, ID 기반 인증 |
| **Level 4** | 완전 봉인, 환경(전압·온도) 공격 탐지·대응 |

## 3. 암호화 시스템 설계 시 고려사항

| 고려 | 내용 |
|---|---|
| **승인 알고리즘** | AES·SHA-2·RSA 등 검증된 알고리즘 |
| **키 관리** | 생성·분배·저장·폐기 전주기 보호 |
| **난수(RNG)** | 승인된 DRBG, 충분한 엔트로피 |
| **자가시험** | 전원·조건부 자가시험(Self-Test) |

## 4. 보안 요소·위협 대응 전략

| 구분 | 내용 |
|---|---|
| **보안 요소** | 물리 보안, 역할·인증, 키 관리, 자가시험, EMI/EMC |
| **물리 공격** | 변조 증거·대응(봉인·제로화) |
| **부채널 공격** | 전력·타이밍 분석 방어(마스킹) |
| **키 노출** | HSM·키 제로화, 접근통제 |

## 5. 고려사항 및 시사점
- 후속 표준 **FIPS 140-3**(ISO/IEC 19790 기반)으로 전환
- 국내는 **KCMVP**(검증 암호모듈)로 대응
- 조달·컴플라이언스 요건, 양자내성암호(PQC) 대비

---

> **한 줄 요약**: FIPS 140-2는 *암호모듈 보안 요구사항 표준* 으로 Level 1~4(물리보호·인증 강도별)로 분류하며, 승인 알고리즘·키 관리·자가시험·물리/부채널 대응을 설계 요소로 요구한다.

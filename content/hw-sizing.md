# 정보시스템 하드웨어 규모산정 지침 (TTAK.KO-10.0292/R3)

## 1. 개요

### 가. 정의
> 정보시스템 구축 시 서버·스토리지·네트워크 등 **하드웨어 용량을 객관적·정량적 기준으로 산정**하기 위한 TTA 표준 지침(2023.12.06 R3 개정).

### 나. 필요성
- **과다 산정**(예산 낭비) / **과소 산정**(성능 저하) 방지
- 공공·대규모 사업의 **객관적 근거·감리** 확보

### 다. 규모산정 대상
- **서버(WAS/DB/AP)**, **스토리지·백업**, **네트워크**, **보안장비**

## 2. 규모산정 절차

```mermaid
flowchart LR
  A[요구·업무량 분석] --> B[산정 대상·방식 선정]
  B --> C[기준값·보정계수 적용]
  C --> D[규모 산출]
  D --> E[검증·조정]
```

| 단계 | 내용 |
|---|---|
| **업무량 분석** | 동시사용자·트랜잭션(TPS)·데이터량·피크 파악 |
| **방식 선정** | 대상별 산정 방식(정량·참조) 결정 |
| **보정 적용** | 성능·여유율·이중화·가용률 계수 반영 |
| **산출·검증** | 규모 계산 후 타당성·벤치마크 검증 |

## 3. 규모산정 방식

| 방식 | 설명 |
|---|---|
| **수치(정량) 기반** | TPC·SPEC 등 **성능지표(tpmC·OPS·SPECint)** 로 계산 |
| **참조 모델 기반** | 유사 시스템 사례·표준 구성 비교 |
| **보정·여유율** | 피크율·증가율·이중화·목표가용성 반영 |

- 예: 필요 성능 = 기준 트랜잭션량 × 피크율 × 여유율 ÷ 장비 단위성능

## 4. 고려사항

| 구분 | 내용 |
|---|---|
| **피크·성장** | 피크 부하, 향후 3~5년 증설 반영 |
| **가상화·클라우드** | 자원 공유·오토스케일 환경 반영 |
| **가용성** | 이중화(N+1)·재해복구 용량 |
| **검증** | 벤치마크(BMT)·부하시험으로 실증 |

## 5. 시사점
- 클라우드 전환 시 **온디맨드·오토스케일**로 사이징 패러다임 변화
- 그럼에도 **초기 용량·비용 산정**의 객관 근거로 표준 지침 유효

---

> **한 줄 요약**: TTAK.KO-10.0292/R3은 *업무량 분석→방식 선정→보정 적용→산출·검증* 절차로 서버·스토리지 등 HW 용량을 성능지표(tpmC 등) 기반으로 객관 산정하는 표준 지침이다.

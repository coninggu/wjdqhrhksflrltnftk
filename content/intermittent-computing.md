# 인터미턴트 컴퓨팅(Intermittent Computing)

## 1. 개요

### 가. 정의
> 배터리 없이 **주변 에너지 하베스팅(Energy Harvesting)** 으로 동작하는 장치가, 전력이 불안정하게 공급·차단되는 환경에서도 **연산 상태를 보존·복구하며 작업을 이어가는** 컴퓨팅 방식.

### 나. 등장 배경
- 배터리리스 IoT·초소형 센서 확산 → **배터리 교체·수명 한계** 극복
- 태양광·RF·진동 등 수확 에너지는 **간헐적·불규칙** → 잦은 정전 대응 필요

## 2. 동작 원리

```mermaid
flowchart LR
  H[에너지 수확<br/>태양광·RF·진동] --> B[커패시터 충전]
  B --> E[연산 실행]
  E --> C[체크포인트 저장<br/>비휘발성 메모리]
  C -. 전력 차단 .-> P((정전))
  P -. 전력 회복 .-> R[상태 복구]
  R --> E
```

## 3. 핵심 기술 요소

| 요소 | 설명 |
|---|---|
| **에너지 하베스팅** | 태양광·RF·열·진동 등 주변 에너지 수집·저장 |
| **비휘발성 메모리(NVM)** | FRAM·MRAM으로 전원 차단 시 상태 보존 |
| **체크포인팅(Checkpointing)** | 연산 중간 상태를 주기적/적응적으로 저장 |
| **멱등성(Idempotency)** | 재실행 시에도 결과 일관성 보장 |
| **전력 인식 스케줄링** | 가용 에너지에 맞춘 작업 분할·실행 |

## 4. 주요 이슈

| 이슈 | 내용 |
|---|---|
| **진행 보장(Forward Progress)** | 체크포인트 오버헤드 vs 재실행 손실 균형 |
| **일관성** | 정전 시점 메모리 불일치(비원자성) 방지 |
| **성능** | 잦은 저장/복구로 인한 처리 지연 |
| **디버깅** | 비결정적 정전으로 재현·검증 곤란 |

## 5. 적용 분야

| 분야 | 활용 |
|---|---|
| **배터리리스 IoT** | 환경·구조물 모니터링 센서 |
| **웨어러블·헬스** | 초저전력 생체 센서 |
| **엣지 AI** | TinyML 결합 초저전력 추론 |

## 6. 고려사항 및 시사점
- 체크포인팅 최적화(적응형·차등 저장)가 성능의 관건
- **지속가능(Green)·유지보수리스** 컴퓨팅 실현
- NVM·초저전력 MCU 발전으로 상용화 확대

---

> **한 줄 요약**: 인터미턴트 컴퓨팅은 *에너지 하베스팅 기반 간헐 전력* 환경에서 **체크포인팅·비휘발성 메모리** 로 상태를 보존·복구하며 작업을 이어가는 초저전력 컴퓨팅으로, 배터리리스 IoT·엣지 AI에 활용된다.

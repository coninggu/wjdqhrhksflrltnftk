# 정보시스템 마스터플랜(ISMP)

## 1. 개요

### 가. 정의
> 특정 정보시스템 **구축 사업의 명확한 요구사항을 사전에 도출·기획**하여, 사업 발주에 필요한 제안요청서(RFP)와 실행계획을 수립하는 활동. ISP가 전사 정보화 전략이라면 ISMP는 **개별 사업 단위의 상세 기획**이다.

ISMP가 도입된 배경은 '**모호한 요구로 발주된 SW 사업의 실패**'였다. 요구가 불명확한 채 사업이 시작되면 과업 변경·분쟁이 잦다. ISMP는 발주 전에 요구사항을 상세히 분석·확정해 사업 리스크를 줄이고 발주 품질을 높인다. (SW진흥법상 일정 규모 이상 사업에 권고)

## 2. ISP와 비교

| 구분 | ISP | ISMP |
|---|---|---|
| **범위** | 전사 정보화 전략 | 개별 시스템 사업 |
| **목적** | 중장기 정보화 방향 | 사업 요구·발주 상세 기획 |
| **산출물** | 정보화 마스터플랜 | RFP·요구사항 정의서 |

## 3. 단계별 활동·산출물

```mermaid
flowchart LR
  A[프로젝트 착수·계획] --> B[정보시스템 방향성 수립] --> C[업무·기술 요건 분석] --> D[정보시스템 구조·요건 정의] --> E[사업 이행방안 수립]
  style C fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 단계 | 세부 활동 | 산출물 |
|---|---|---|
| **착수·계획** | 범위·추진체계 정의 | 수행계획서 |
| **방향성 수립** | 환경·현황 분석, 목표모델 | 방향성 정의서 |
| **요건 분석** | 업무·기술 요구 상세 분석 | 요구사항 분석서 |
| **구조·요건 정의** | 기능·데이터·아키텍처 정의 | 요구사항 정의서 |
| **이행방안** | 규모·비용 산정, RFP 작성 | 제안요청서(RFP), 이행계획 |

## 4. 시사점
- 발주 전 **요구사항 상세화**로 과업 변경·분쟁 예방
- 정확한 규모·비용 산정으로 적정 대가 확보
- ISP(전략)→ISMP(사업기획)→구축의 연계로 정합성 확보

---

> **한 줄 요약**: ISMP는 *개별 정보시스템 사업의 요구사항을 발주 전 상세 기획* 하는 활동으로, 착수→방향성→요건분석→구조정의→이행방안 단계를 거쳐 RFP·요구사항 정의서를 산출해 사업 리스크를 줄인다.

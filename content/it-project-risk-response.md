# IT 프로젝트 리스크 대응(Risk Response)

## 1. 개요

### 가. 정의
> 프로젝트 목표를 위협하거나 기회가 되는 **불확실성(리스크)을 식별·분석하고 대응 전략을 수립·실행**하는 관리 활동. PMBOK 리스크 관리 지식영역의 핵심.

### 나. 필요성
- IT 프로젝트의 높은 불확실성(요구변경·기술·일정) 대응
- 위협 최소화 + **기회 극대화**

## 2. 리스크 대응 계획 수립 절차 (가)

```mermaid
flowchart LR
  A[리스크 식별] --> B[정성적 분석<br/>발생확률·영향] --> C[정량적 분석] --> D[대응 계획 수립] --> E[대응 실행·통제]
  E -. 재평가 .-> A
  style D fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 단계 | 내용 |
|---|---|
| **식별** | 리스크 목록(Register) 작성 |
| **정성 분석** | 발생확률·영향도 평가(P-I Matrix), 우선순위화 |
| **정량 분석** | 금액·일정 영향 정량화(EMV, 몬테카를로) |
| **대응 계획** | 리스크별 전략·담당자·예비비(Contingency) 지정 |
| **실행·통제** | 트리거 모니터링, 잔여·2차 리스크 관리 |

## 3. 위협(부정적)에 대한 대응 전략 (나)

| 전략 | 내용 | 예 |
|---|---|---|
| **회피(Avoid)** | 위협 원인 제거 | 리스크 큰 범위 삭제 |
| **전가(Transfer)** | 제3자에게 이전 | 보험·아웃소싱·계약 |
| **완화(Mitigate)** | 확률·영향 감소 | 프로토타입·이중화 |
| **수용(Accept)** | 대응 없이 감수 | 예비비 확보(소규모 리스크) |

## 4. 기회(긍정적)에 대한 대응 전략 (다)

| 전략 | 내용 | 예 |
|---|---|---|
| **활용(Exploit)** | 기회 확실히 실현 | 우수 인력 우선 투입 |
| **공유(Share)** | 제3자와 협력해 실현 | 파트너십·조인트벤처 |
| **증대(Enhance)** | 발생확률·영향 증가 | 자원 추가 투입 |
| **수용(Accept)** | 적극 조치 없이 기회 활용 | 발생 시 이득 수취 |

## 5. 시사점
- 위협·기회 모두 관리 대상 — **에스컬레이션**(권한 밖 리스크는 상위 이관)
- 예비비(Contingency/Management Reserve)로 재무적 대비
- 애자일에선 반복적 리스크 재평가(스프린트 단위)

---

> **한 줄 요약**: 리스크 대응은 *식별→정성/정량 분석→대응계획→실행통제* 절차로, 위협은 *회피·전가·완화·수용*, 기회는 *활용·공유·증대·수용* 전략으로 관리한다.

# 미라이 봇넷(Mirai Botnet)

## 1. 개요

### 가. 개념
> **미라이 봇넷**은 보안이 취약한 **IoT 기기(IP카메라·공유기 등)를 대량 감염시켜 좀비로 만든 뒤, 대규모 DDoS 공격에 악용한 악성코드·봇넷**이다. 2016년 대규모 인터넷 마비 사태를 일으킨 대표적 IoT 위협이다.

미라이가 IoT 보안의 경종을 울린 근본 이유는 '**허술한 IoT 기기 하나하나가 강력한 공격 무기가 된다**'는 것을 실증했다는 데 있다. 미라이는 정교한 취약점 공격이 아니라, 놀랍도록 단순한 방법을 썼다. 인터넷에 연결된 IoT 기기를 스캔해, 공장 출하 시 설정된 **기본 계정·비밀번호**(admin/admin 같은)를 그대로 쓰는 기기에 로그인해 감염시켰다. 수많은 IoT 기기가 비밀번호를 바꾸지 않은 채 방치돼 있었기에, 미라이는 수십만 대를 순식간에 좀비로 만들었다. 이 좀비들이 일제히 특정 대상에 트래픽을 퍼부어(DDoS), 2016년 DNS 서비스 업체 Dyn을 마비시켜 트위터·넷플릭스 등 주요 사이트를 접속 불능으로 만들었다. 미라이는 IoT가 편의를 주는 만큼 보안 사각지대가 되어, 개별 기기의 허술함이 인터넷 전체를 위협할 수 있음을 보여준 사건이다.

### 나. 특징
| 특징 | 내용 |
|---|---|
| **공격 방식** | 기본 계정·비밀번호 사전 대입으로 IoT 감염 |
| **목적** | 대규모 DDoS 공격 |
| **규모** | 수십만 대 좀비 IoT 동원 |
| **파급** | 소스코드 공개로 변종 다수 확산 |

## 2. IoT 서비스 생애주기별 보안위협 및 대응

IoT 보안은 특정 시점이 아니라 기기의 전 생애주기에 걸쳐 확보해야 한다. 미라이는 '설치 시 기본 비밀번호 방치'와 '운영 중 업데이트 부재'라는 생애주기 허점을 파고들었다.

| 단계 | 보안위협 | 대응 |
|---|---|---|
| **설계·개발** | 취약한 설계, 하드코딩 계정 | 시큐어 설계·코딩, 기본 보안 내장 |
| **배포·설치** | 기본 비밀번호, 불필요 포트 개방 | 초기 비밀번호 강제 변경, 최소 노출 |
| **운영·이용** | 미패치 취약점, 악성코드 감염 | 정기 펌웨어 업데이트, 이상 탐지 |
| **폐기** | 잔존 데이터·계정 유출 | 데이터 완전 삭제, 인증정보 폐기 |

## 3. IoT 공통보안 7대 원칙

IoT 보안 강화를 위해 정부·기관이 제시한 공통보안 원칙은 설계부터 폐기까지 보안을 내재화할 것을 요구한다.

| 원칙(요지) | 내용 |
|---|---|
| **1. 설계 단계 보안 내재화** | 기획·설계부터 보안 반영(Security by Design) |
| **2. 안전한 SW·하드웨어** | 검증된 개발·시큐어 코딩 |
| **3. 안전한 초기 보안설정** | 기본 비밀번호 변경 등 안전 기본값 |
| **4. 인증·암호화 적용** | 상호 인증, 데이터 암호화 |
| **5. 최신 보안 패치·업데이트** | 지속적 취약점 조치 |
| **6. 안전한 운영·관리** | 이상 탐지, 접근 통제 |
| **7. 침해 대응·복구 체계** | 사고 대응·안전한 폐기 |

## 4. 고려사항 및 시사점

1. **기본값 보안이 첫 방어선**이다. 미라이가 증명했듯 기본 계정·비밀번호 방치가 최대 취약점이므로, 설치 시 강제 비밀번호 변경과 안전한 초기 설정이 필수다.
2. **생애주기 전반의 보안 관리**가 필요하다. IoT는 배포 후 오래 방치되기 쉬우므로, 지속적 펌웨어 업데이트·원격 관리 체계로 운영 중 보안을 유지해야 한다.
3. **집단 위협의 관점**이 중요하다. IoT 기기는 개별로는 사소해도 대량 동원되면 인터넷 인프라를 위협하므로, 제조사·이용자·통신사가 함께 책임지는 생태계 차원의 보안이 요구된다.

---

> **한 줄 요약**: 미라이 봇넷은 *기본 비밀번호가 방치된 IoT 기기를 대량 감염시켜 DDoS에 악용* 한 대표적 IoT 위협으로, IoT 생애주기별 보안(안전한 초기설정·지속 업데이트)과 공통보안 7대 원칙(설계부터 보안 내재화)의 필요성을 일깨웠다.

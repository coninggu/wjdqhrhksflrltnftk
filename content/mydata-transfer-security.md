# 마이데이터 전송 보안 (마이데이터 전송 보안 안내서, 2023.09)

## 1. 개요

### 가. 배경
> 마이데이터(본인신용정보관리업) 확산으로 개인정보 **전송 과정의 안전성** 확보가 중요해짐. 개인정보보호위원회가 전송 보안 안내서를 발간.

### 나. 목적
- 전송자·수신자의 개인정보 **안전한 처리·전송** 보장

## 2. 전송대상 개인정보 보호책임자 지정

| 항목 | 내용 |
|---|---|
| **CPO 지정** | 전송대상 개인정보 **보호책임자(CPO)** 지정·책임 명확화 |
| **역할** | 전송 보안 정책 수립·점검, 사고 대응 총괄 |
| **독립성** | 권한·독립성 부여, 경영진 보고 체계 |

## 3. 전송대상 개인정보처리시스템 접근 관리

```mermaid
flowchart LR
  A[접근 권한 최소화] --> B[인증 강화]
  B --> C[접속기록 관리]
  C --> D[이상행위 탐지]
```

| 항목 | 내용 |
|---|---|
| **접근권한** | 최소권한·직무분리, 권한 부여·회수 관리 |
| **인증** | 안전한 인증수단(다요소), 계정 관리 |
| **접속기록** | 접속기록 보관·점검(위·변조 방지) |
| **통제** | 접근통제 시스템, 이상행위 모니터링 |

## 4. 개인정보 관리 및 재해·재난 대비

| 항목 | 내용 |
|---|---|
| **암호화** | 전송구간(TLS)·저장 암호화 |
| **무결성·유출방지** | 위·변조 방지, 유출 탐지·차단 |
| **재해·재난 대비** | 백업·복구 체계(BCP/DRS), 이중화 |
| **사고 대응** | 침해사고 대응 절차·신고 |

## 5. 고려사항 및 시사점
- **전송 전 구간의 안전성**(인증·암호화·접근통제)이 핵심
- 개인정보보호법·신용정보법 준수, 정기 점검·교육
- API 기반 전송의 보안(인증·mTLS)과 연계

---

> **한 줄 요약**: 마이데이터 전송 보안 안내서는 *보호책임자(CPO) 지정, 처리시스템 접근관리(최소권한·인증·접속기록), 개인정보 관리·재해대비(암호화·백업·사고대응)* 로 전송 개인정보의 안전한 처리를 요구한다.

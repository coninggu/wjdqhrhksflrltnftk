# NoSQL 유형과 데이터 모델링 절차

## 1. 개요

### 가. 정의
> 관계형 스키마·SQL·강한 정합성에서 벗어나 **대용량·비정형·분산·고확장성**에 최적화된 데이터 저장 기술(Not Only SQL).

### 나. 특징
- **스키마리스**, **수평 확장(Scale-out)**, **BASE**(Eventually Consistent), CAP 이론 기반 선택

## 2. NoSQL 유형

| 유형 | 모델 | 예시 | 용도 |
|---|---|---|---|
| **Key-Value** | 키-값 쌍 | Redis, DynamoDB | 캐시·세션 |
| **Document** | JSON 문서 | MongoDB | 반정형 데이터 |
| **Column-Family** | 컬럼 지향 | Cassandra, HBase | 대용량 시계열 |
| **Graph** | 노드-간선 | Neo4j | 관계망·추천 |

## 3. 데이터 모델링 절차

```mermaid
flowchart LR
  A[요구·쿼리 분석] --> B[접근 패턴 정의]
  B --> C[집계·비정규화 설계]
  C --> D[키·인덱스 설계]
  D --> E[검증·튜닝]
```

| 단계 | 내용 |
|---|---|
| **쿼리 우선(Query-First)** | 조회 패턴을 먼저 분석 (RDB와 반대) |
| **비정규화·집계** | Join 회피 위해 데이터 중복·임베딩 |
| **키 설계** | 파티션·정렬 키로 분산·성능 최적화 |

## 4. 고려사항
- **정합성 vs 가용성** 트레이드오프(CAP), 접근 패턴 변경 시 재설계 부담

---

> **한 줄 요약**: NoSQL은 *비정형·대용량·분산에 최적화된 저장 기술* 로 Key-Value·Document·Column·Graph 유형이 있고, 조회 패턴을 먼저 분석하는 쿼리 우선 비정규화 모델링을 따른다.

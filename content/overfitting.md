# 과적합(Overfitting)의 발생 이유와 해결 방안

## 1. 개요

### 가. 정의
> **과적합(Overfitting)** 은 머신러닝 모델이 **학습 데이터에 지나치게 맞춰져, 새로운 데이터(테스트·실제)에 대한 예측 성능이 떨어지는** 현상이다. 모델이 데이터의 일반적 패턴이 아니라 노이즈·특수성까지 외워버린 상태다.

과적합의 본질은 '**외우기와 이해하기의 차이**'다. 학습의 진짜 목표는 데이터에 담긴 일반적 규칙을 배워 처음 보는 데이터도 잘 예측하는 것(일반화)이다. 그런데 모델이 너무 복잡하거나 학습이 과하면, 규칙을 이해하는 대신 학습 데이터를 통째로 암기해버린다. 시험 문제의 원리를 이해한 학생과 답만 외운 학생의 차이와 같다. 답만 외운 학생은 본 문제(학습 데이터)는 완벽히 풀지만 조금만 바뀐 문제(새 데이터)에는 무너진다. 과적합된 모델도 학습 데이터에서는 정확도가 매우 높지만 실제 데이터에서는 성능이 급락한다. 반대로 모델이 너무 단순해 패턴조차 못 배우면 **과소적합(Underfitting)** 이 된다. 좋은 모델은 이 둘 사이의 균형점, 즉 일반화가 잘 되는 지점에 있다.

## 2. 해결 방안

| 방안 | 내용 |
|---|---|
| **데이터 증강·확보** | 더 많고 다양한 데이터로 일반화 향상 |
| **정규화(Regularization)** | L1·L2로 가중치 크기 제한(복잡도 억제) |
| **드롭아웃(Dropout)** | 학습 시 일부 뉴런 무작위 제거 |
| **조기 종료(Early Stopping)** | 검증 성능 저하 시 학습 중단 |
| **교차검증** | 여러 분할로 일반화 성능 확인 |
| **모델 단순화** | 파라미터·층 축소, 특징 선택 |
| **앙상블** | 여러 모델 결합으로 분산 감소 |

핵심 전략은 '**모델 복잡도를 낮추거나(정규화·드롭아웃·단순화) 데이터를 늘리는(증강)** 것'이다. 그리고 검증 데이터로 일반화 성능을 지속 확인해, 과적합 조짐이 보이면 학습을 멈춘다(조기 종료).

## 3. 편향-분산 트레이드오프

과적합은 **분산(Variance)이 큰** 상태(데이터 변화에 민감), 과소적합은 **편향(Bias)이 큰** 상태(패턴을 못 배움)다. 좋은 모델은 편향과 분산이 균형을 이뤄 총 오차가 최소가 되는 지점에 있다.

## 4. 고려사항 및 시사점

1. **검증 데이터로 과적합을 진단**한다. 학습 정확도는 높은데 검증 정확도가 낮으면 과적합이므로, 학습·검증 성능의 격차를 모니터링해 조기에 대응한다.
2. **데이터 확보가 근본 해법**이다. 정규화·드롭아웃도 효과적이지만, 대표성 있는 데이터를 충분히 확보하는 것이 과적합을 막는 가장 확실한 방법이다.
3. **딥러닝에서 특히 중요**하다. 파라미터가 방대한 딥러닝은 과적합에 취약하므로, 드롭아웃·데이터 증강·전이학습·조기 종료를 조합해 일반화 성능을 확보한다.

---

> **한 줄 요약**: 과적합은 *모델이 학습 데이터를 외워 새 데이터에 성능이 떨어지는* 현상으로, 모델 복잡도 과다·데이터 부족·노이즈·과도한 학습이 원인이며, 정규화·드롭아웃·조기종료·데이터 증강·교차검증으로 일반화 성능을 확보한다.

# 정보시스템 성능 요구사항의 주요 성능지표

## 1. 개요

### 가. 정의
> 정보시스템이 충족해야 할 **성능 목표를 정량적으로 정의**한 비기능 요구사항. 설계·구축·검수·SLA의 기준이 된다.

### 나. 필요성
- 모호한 "빠르게"가 아닌 **측정 가능·검증 가능한 기준** 필요
- 용량산정·아키텍처·부하시험의 근거

## 2. 주요 성능지표

```mermaid
flowchart LR
  R[응답시간] --- T[처리량 TPS]
  T --- C[동시사용자]
  C --- U[자원사용률]
```

| 지표 | 내용 |
|---|---|
| **응답시간(Response Time)** | 요청~응답 완료 시간(예: 3초 이내) |
| **처리량(Throughput/TPS)** | 단위시간당 처리 건수(초당 트랜잭션) |
| **동시 사용자 수** | 동시 접속·처리 가능 사용자 |
| **자원 사용률** | CPU·메모리·디스크·네트워크 사용률(임계치) |
| **가용성** | 가동률(예: 99.9%), MTBF/MTTR |
| **확장성** | 부하 증가 시 성능 유지·확장 능력 |

## 3. 작성 시 고려사항

| 고려 | 내용 |
|---|---|
| **측정 조건 명시** | 피크/평시, 데이터량, 동시성 조건 |
| **정량·검증 가능** | 수치·단위·목표치, 측정 방법 |
| **피크 부하 기준** | 최대 부하 시점 기준 산정 |
| **SLA 연계** | 계약 서비스 수준과 일치 |

## 4. 검증 방법

| 방법 | 내용 |
|---|---|
| **성능 테스트** | 부하·스트레스·내구성 테스트 |
| **BMT** | 벤치마크 테스트로 목표 검증 |
| **모니터링(APM)** | 운영 중 지표 상시 측정 |

## 5. 고려사항 및 시사점
- 성능 요구는 **용량산정·아키텍처 설계**와 직결
- 목표 미달 시 병목 분석·튜닝(스케일업/아웃)
- 클라우드 오토스케일·관측성으로 성능 관리 자동화

---

> **한 줄 요약**: 성능 요구사항은 *응답시간·처리량(TPS)·동시사용자·자원사용률·가용성* 등을 피크 부하·측정조건과 함께 정량·검증 가능하게 정의하며, 성능 테스트·BMT·APM으로 검증한다.

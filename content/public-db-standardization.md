# 공공데이터베이스 표준화 관리 (예방적 품질관리, 2023.04)

## 1. 개요

### 가. 배경
> 행정안전부가 **고품질 공공데이터 제공·활용**을 위해 「공공데이터베이스 표준화 관리 매뉴얼」로 **예방적(사전) 품질관리 기준**을 제시.

### 나. 필요성
- 사후 정제 비용·오류 확산 방지 → **구축 단계부터 품질 내재화**

## 2. 시스템 구축 단계별 예방적 품질관리 활동

```mermaid
flowchart LR
  A[분석] --> D[설계] --> I[구현] --> T[시험/이행] --> O[운영]
```

| 단계 | 품질관리 활동 |
|---|---|
| **분석** | 데이터 요구·표준 정의, 표준단어·도메인 사전 |
| **설계** | 표준 준수 데이터 모델링, 무결성 규칙 설계 |
| **구현** | 표준 반영 DB 구축, 제약조건 적용 |
| **시험·이행** | 품질 진단·검증, 데이터 정제·이관 검증 |
| **운영** | 지속 품질 모니터링·개선 |

## 3. 예방적 품질관리 4개 진단영역·9개 진단항목

| 진단영역 | 진단항목(예) |
|---|---|
| **표준화** | 표준단어, 표준도메인, 표준용어 |
| **모델 품질** | 데이터 모델 정합성, 명명 규칙 |
| **값 품질** | 필수값·유효값(도메인) 준수 |
| **구조·무결성** | 참조 무결성, 코드 정합성 |

> 4개 영역 아래 총 **9개 진단항목**으로 사전 진단(표준단어·도메인·용어, 모델 정합성, 명명, 필수·유효값, 참조무결성, 코드 등).

## 4. 기대 효과

| 효과 | 내용 |
|---|---|
| **품질 향상** | 오류·중복 사전 제거, 정합성 확보 |
| **비용 절감** | 사후 정제 비용 감소 |
| **활용성** | 개방·연계 데이터 신뢰성·활용도↑ |

## 5. 고려사항 및 시사점
- **데이터 표준화·거버넌스** 체계와 연계
- 공공데이터 개방·마이데이터·AI 학습데이터 품질 기반
- 자동 진단 도구·품질관리 조직 운영

---

> **한 줄 요약**: 공공DB 표준화 관리 매뉴얼은 *구축 단계(분석~운영)별 예방적 품질관리 활동* 과 *표준화·모델·값·구조 무결성의 4개 진단영역 9개 진단항목* 으로 사전 품질을 확보한다.

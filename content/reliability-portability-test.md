# 소프트웨어 테스트 종류와 신뢰성·이식성 테스트

## 1. 개요
> 정보시스템 **개발·운영 단계**에서 결함을 발견하고 품질을 확인하기 위해 다양한 테스트가 수행된다. 특히 **ISO/IEC 25010 품질특성** 관점의 **신뢰성·이식성** 테스트는 운영 안정성과 환경 적응성을 보장한다.

## 2. 소프트웨어 테스트의 종류

```mermaid
flowchart LR
  U[단위] --> I[통합] --> S[시스템] --> A[인수]
```

| 분류 기준 | 종류 |
|---|---|
| **테스트 수준** | 단위(Unit)·통합(Integration)·시스템(System)·인수(Acceptance) |
| **테스트 기법** | 블랙박스(명세)·화이트박스(구조)·경험기반 |
| **비기능(품질특성)** | 성능·부하·스트레스·보안·**신뢰성**·**이식성**·사용성·호환성 |
| **변경 대응** | 회귀(Regression)·스모크·확인(Confirmation) |

## 3. 신뢰성 테스트(Reliability Test)
> 명시된 조건에서 **정해진 기간 동안 오류 없이 기능을 지속 수행**하는 능력을 검증.

| 세부 활동(부특성) | 내용·방법 |
|---|---|
| **성숙성(Maturity)** | 결함 발생률·**MTBF**(평균고장간격) 측정, 신뢰성 성장 모델 |
| **결함 허용성(Fault Tolerance)** | 장애·오입력 시에도 지정 성능 유지(이중화·페일오버 시험) |
| **복구성(Recoverability)** | 장애 후 데이터·상태 복구, **RTO/RPO** 검증, 백업·복구 시험 |
| **가용성(Availability)** | 장시간 부하 지속 시험(**Soak/내구성 테스트**) |

## 4. 이식성 테스트(Portability Test)
> 소프트웨어를 **다른 환경(HW·OS·브라우저·플랫폼)** 으로 이전·적응시키는 능력을 검증.

| 세부 활동(부특성) | 내용·방법 |
|---|---|
| **적응성(Adaptability)** | 다양한 환경에 수정 없이 적응하는지 시험 |
| **설치성(Installability)** | 설치·업그레이드·제거의 정상 동작 확인 |
| **대체성(Replaceability)** | 동일 목적의 기존 SW를 대체·호환하는지 |
| **호환성(Co-existence)** | 여러 브라우저·OS·기기 교차 검증(크로스브라우징) |

## 5. 고려사항 및 시사점
- 비기능 테스트는 **정량 목표(SLA·MTBF·RTO)** 기준으로 판정
- 클라우드·멀티플랫폼 환경 확대 → **이식성·호환성** 중요성 증가
- 신뢰성은 **운영 중 지속 모니터링**(APM)과 연계, 카오스 엔지니어링으로 결함 허용성 검증

---

> **한 줄 요약**: 테스트는 수준·기법·비기능·회귀로 분류되며, 신뢰성 테스트는 성숙성·결함허용성·복구성·가용성을, 이식성 테스트는 적응성·설치성·대체성·호환성을 정량 목표 기준으로 검증한다.

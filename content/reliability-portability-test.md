# 소프트웨어 테스트 종류와 신뢰성·이식성 테스트

## 1. 개요
> 개발·운영 단계에서 수행되는 테스트는 목적에 따라 여러 종류로 나뉘며, 특히 **품질 특성(ISO/IEC 25010)** 관점의 신뢰성·이식성 테스트가 중요하다.

## 2. 소프트웨어 테스트 종류

| 분류 | 종류 |
|---|---|
| **수준별** | 단위·통합·시스템·인수 테스트 |
| **기법별** | 블랙박스·화이트박스·경험기반 |
| **목적별(비기능)** | 성능·부하·보안·**신뢰성**·**이식성**·사용성 |
| **변경 대응** | 회귀(Regression) 테스트 |

## 3. 신뢰성 테스트(Reliability)
> 정해진 조건·기간 동안 **오류 없이 지속 동작**하는 능력 검증.

| 세부 활동 | 내용 |
|---|---|
| **성숙성** | 결함 발생률·MTBF 측정 |
| **결함허용성** | 장애 시 정상 동작 유지(Fault Tolerance) |
| **복구성** | 장애 후 데이터·상태 복구 검증 |
| **부하·내구** | 장시간·고부하 안정성(Soak Test) |

## 4. 이식성 테스트(Portability)
> 다른 **환경(OS·HW·브라우저)** 으로 이전·적응하는 능력 검증.

| 세부 활동 | 내용 |
|---|---|
| **적응성** | 다양한 환경 적응 여부 |
| **설치성** | 설치·제거 정상 동작 |
| **대체성** | 기존 SW 대체 호환성 |
| **호환성** | 브라우저·플랫폼 교차 검증 |

---

> **한 줄 요약**: 테스트는 수준·기법·목적별로 나뉘며, 신뢰성 테스트는 성숙성·결함허용성·복구성을, 이식성 테스트는 적응성·설치성·대체성을 세부 활동으로 검증한다.

# SOAR(Security Orchestration, Automation and Response)

## 1. 개요

### 가. 정의 및 등장 배경
> **SOAR**은 보안 위협 탐지부터 대응까지의 과정을 **오케스트레이션(Orchestration)·자동화(Automation)·대응(Response)** 으로 통합해, 보안 운영을 자동화·효율화하는 체계다.

SOAR이 등장한 근본 배경은 '**보안 경보는 폭증하는데 대응할 인력은 부족하다**'는 현실이다. 오늘날 보안관제센터(SOC)에는 수많은 보안 장비에서 하루에도 수천~수만 건의 경보가 쏟아진다. 분석가가 이를 일일이 확인하고, 여러 도구를 오가며 조사하고, 수동으로 대응하기에는 시간과 인력이 턱없이 부족하다. 그 결과 진짜 위협이 경보의 홍수에 묻히거나 대응이 늦어진다. SOAR은 이 문제를 자동화로 푼다. 흩어진 보안 도구들을 하나로 연동하고(오케스트레이션), 반복적인 조사·대응 작업을 미리 정의한 절차(플레이북)에 따라 자동 수행하며(자동화), 위협을 신속히 차단·조치한다(대응). 그러면 분석가는 단순 반복 업무에서 벗어나 정말 중요한 판단에 집중할 수 있고, 대응 속도가 극적으로 빨라진다.

### 나. 필요성
사이버 위협의 양적 폭증과 보안 인력 부족, 여러 보안 도구의 파편화가 맞물리며, 보안 운영을 자동화·통합하는 SOAR이 필수가 되었다.

## 2. 구성요소와 주요 기능

```mermaid

  O["오케스트레이션<br/>(도구 연동)"] --> A["자동화<br/>(플레이북)"] --> R["대응<br/>(차단·조치)"]
  style A fill:#e8f0fe,stroke:#2f6fed,stroke-width:2px
```

| 구성요소 | 내용 |
|---|---|
| **오케스트레이션** | 이기종 보안 도구(SIEM·EDR·방화벽) 연동·통합 |
| **자동화(플레이북)** | 대응 절차를 플레이북으로 정의·자동 실행 |
| **사고 대응(IR)** | 위협 조사·격리·차단, 케이스 관리 |
| **위협 인텔리전스** | 위협 정보 연계로 판단 지원 |
| **대시보드·측정** | 대응 현황·지표(MTTR) 관리 |

핵심은 **플레이북** 이다. "이런 유형의 경보가 오면 → 이렇게 조사하고 → 이렇게 대응하라"는 절차를 코드화해, 사람 개입 없이(또는 최소 승인만으로) 자동 실행한다.

## 3. SIEM과의 관계

| 구분 | SIEM | SOAR |
|---|---|---|
| **초점** | 로그 수집·분석·탐지 | 대응 자동화·오케스트레이션 |
| **역할** | 위협 탐지·경보 | 탐지 후 조사·대응 |
| **관계** | SOAR의 입력(경보 제공) | SIEM 경보를 받아 자동 대응 |

SIEM이 위협을 '탐지'하면, SOAR이 그 경보를 받아 '대응'을 자동화한다. 둘은 상호 보완적으로 함께 쓰인다.

## 4. 기대효과와 도입 시 고려사항

| 구분 | 내용 |
|---|---|
| **기대효과** | 대응 시간(MTTR) 단축, 분석가 업무 경감, 일관된 대응, 24/365 자동 대응 |
| **도입 고려** | 플레이북 설계 품질, 오탐 시 자동 대응 위험(승인 절차), 도구 연동 표준화, 인력 역량 |

## 5. 고려사항 및 시사점

1. **플레이북 품질이 성패를 좌우**한다. 자동화의 효과는 플레이북에 달렸으므로, 정확하고 검증된 대응 절차를 설계하고 지속 개선해야 한다. 잘못된 플레이북은 오히려 피해를 키운다.
2. **완전 자동화와 사람 개입의 균형**이 필요하다. 오탐에 자동 대응하면 정상 서비스를 차단할 수 있으므로, 중대한 조치는 사람의 승인을 거치는 반자동(Human-in-the-loop) 설계가 안전하다.
3. **AI 결합으로 지능화**된다. 생성형 AI·머신러닝을 결합해 위협 분석·플레이북 추천·자동 조사를 고도화하는 방향으로 발전하며, 보안 운영의 자율화를 이끈다.

---

> **한 줄 요약**: SOAR은 *오케스트레이션·자동화·대응* 으로 보안 운영을 통합 자동화하는 체계로, 플레이북으로 반복 대응을 자동 실행해 대응시간(MTTR)을 단축하며, SIEM(탐지)과 상호 보완하되 플레이북 품질과 사람 개입 균형이 관건이다.

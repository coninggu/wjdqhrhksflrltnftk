# 정렬 알고리즘(버블·삽입·퀵 정렬)

## 1. 개요

### 가. 정의
> **정렬 알고리즘**은 주어진 데이터 집합을 사용자가 지정한 기준(오름/내림차순)에 맞게 순서대로 재배치하는 기법. 탐색·병합 등 다른 연산의 효율을 좌우하는 기초 알고리즘이다.

정렬 알고리즘을 이해하는 핵심은 '**시간 복잡도와 안정성·추가 메모리의 트레이드오프**'다. 단순한 알고리즘(버블·삽입)은 구현이 쉽지만 O(n²)로 느리고, 분할정복 기반(퀵·병합)은 평균 O(n log n)로 빠르다. 데이터 크기·정렬 상태·메모리 제약에 따라 적합한 알고리즘이 달라진다.

## 2. 버블 정렬 (가)
> 인접한 두 원소를 비교해 순서가 잘못되면 교환하는 과정을 반복하며, 큰 값이 거품처럼 뒤로 이동한다.
- **동작**: 한 회전마다 최댓값이 맨 뒤로 확정, n-1회 반복
- **복잡도**: 시간 O(n²), 공간 O(1), **안정 정렬**
- 특징: 구현 단순하나 매우 비효율, 교육용

## 3. 삽입 정렬 (나)
> 정렬된 부분에 새 원소를 **올바른 위치에 끼워 넣는** 방식. 카드를 손에 정렬하는 방식과 유사하다.
- **동작**: 두 번째 원소부터 앞의 정렬된 구간과 비교해 삽입
- **복잡도**: 시간 O(n²)(최선 O(n)), 공간 O(1), **안정 정렬**
- 특징: **거의 정렬된 데이터·소규모**에서 매우 효율적

## 4. 퀵 정렬 (다)
> **피벗(pivot)** 을 기준으로 작은 값과 큰 값으로 분할하고, 각 부분을 재귀적으로 정렬하는 **분할정복** 알고리즘.
- **동작**: 피벗 선택 → 분할(partition) → 좌·우 재귀
- **복잡도**: 평균 O(n log n), **최악 O(n²)**(이미 정렬·나쁜 피벗), 공간 O(log n), 불안정
- 특징: 평균적으로 가장 빠른 정렬, 제자리(in-place)

## 5. 비교

| 알고리즘 | 평균 | 최악 | 공간 | 안정성 |
|---|---|---|---|---|
| **버블 정렬** | O(n²) | O(n²) | O(1) | 안정 |
| **삽입 정렬** | O(n²) | O(n²) | O(1) | 안정 |
| **퀵 정렬** | O(n log n) | O(n²) | O(log n) | 불안정 |

## 6. 시사점
- 퀵 정렬은 최악 O(n²)를 피하려 **피벗 선택(랜덤·중앙값)** 최적화
- 안정성 필요 시 병합 정렬, 실무 라이브러리는 **하이브리드(팀소트·인트로소트)** 사용
- 데이터 특성(크기·정렬도·안정성 요구)에 맞는 선택이 핵심

---

> **한 줄 요약**: 버블·삽입 정렬은 *O(n²)의 단순 정렬*(삽입은 거의 정렬된 데이터에 효율적), 퀵 정렬은 *평균 O(n log n)의 분할정복* 으로 빠르나 최악 O(n²)이며, 데이터 특성에 맞게 선택한다.

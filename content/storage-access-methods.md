# 블록·파일·오브젝트 스토리지의 데이터 접근방식

## 1. 개요

### 가. 정의
> 데이터 저장·접근 단위에 따라 스토리지를 **블록(Block)·파일(File)·오브젝트(Object)** 로 구분하며, 각기 접근방식·용도가 다르다.

### 나. 필요성
- 워크로드 특성(성능·공유·확장·비용)에 맞는 스토리지 선택

## 2. 접근방식 비교

```mermaid
flowchart TB
  B[블록 스토리지<br/>블록 단위·SAN] 
  F[파일 스토리지<br/>파일/디렉터리·NAS]
  O[오브젝트 스토리지<br/>객체+메타데이터·HTTP]
```

| 구분 | 블록 | 파일 | 오브젝트 |
|---|---|---|---|
| **접근 단위** | 고정 크기 블록(LBA) | 파일(경로/계층) | 객체(고유 ID+메타데이터) |
| **인터페이스** | iSCSI·FC(SAN) | NFS·SMB(NAS) | REST API(HTTP) |
| **구조** | 논리 볼륨 | 계층적 디렉터리 | 평면적(Flat) 네임스페이스 |
| **성능** | 최고(저지연) | 중간 | 상대적 낮음(고지연) |
| **확장성** | 제한적 | 중간 | 매우 높음(무한 확장) |
| **용도** | DB·VM·트랜잭션 | 파일 공유·협업 | 백업·아카이브·미디어·빅데이터 |

## 3. 특징 상세

| 유형 | 특징 |
|---|---|
| **블록** | OS가 파일시스템 구성, 고성능·저지연, 공유 제한 |
| **파일** | 파일시스템 그대로 공유, 다중 접근·POSIX |
| **오브젝트** | 메타데이터 풍부, HTTP 접근, 대규모·저비용(예: S3) |

## 4. 선택 기준

| 요구 | 적합 |
|---|---|
| **고성능 트랜잭션** | 블록 |
| **다중 사용자 파일 공유** | 파일 |
| **대규모 비정형·아카이브** | 오브젝트 |

## 5. 고려사항 및 시사점
- 클라우드에서 **EBS(블록)·EFS(파일)·S3(오브젝트)** 로 대응
- 오브젝트는 **불변성·버전관리·수명주기**로 비용 최적화
- 워크로드별 혼합(하이브리드) 구성이 일반적

---

> **한 줄 요약**: 블록은 *블록 단위(SAN)로 고성능 DB·VM*, 파일은 *파일 단위(NAS)로 공유*, 오브젝트는 *객체+메타데이터(HTTP)로 대규모 아카이브* 에 적합하며, 접근 단위·인터페이스·확장성이 다르다.

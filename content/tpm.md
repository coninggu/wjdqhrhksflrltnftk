# TPM(Trusted Platform Module)

## 1. 개요

### 가. 정의
> 암호키 생성·저장, 무결성 측정 등을 수행하는 **하드웨어 기반 보안 칩(보안 코프로세서)**. TCG(Trusted Computing Group) 표준(TPM 2.0).

### 나. 필요성
- 소프트웨어만으로 부족한 **하드웨어 신뢰 기반(Root of Trust)** 제공
- 키 탈취·부팅 조작 방지

## 2. 주요 기능

```mermaid
flowchart LR
  K[키 생성·보관] --- M[무결성 측정<br/>PCR]
  M --- S[봉인 Sealing]
  S --- A[원격 검증<br/>Attestation]
```

| 기능 | 설명 |
|---|---|
| **키 생성·보관** | 하드웨어 내부에 개인키 격리 저장 |
| **무결성 측정** | 부팅 구성요소 해시를 **PCR**에 기록 |
| **봉인(Sealing)** | 특정 플랫폼 상태에서만 복호 가능하게 데이터 암호화 |
| **원격 검증(Attestation)** | 플랫폼 무결성을 원격 증명 |
| **난수 생성(RNG)** | 하드웨어 기반 엔트로피 |

## 3. 활용

| 분야 | 활용 |
|---|---|
| **보안 부팅** | Measured/Secure Boot 신뢰 체인 |
| **디스크 암호화** | BitLocker 키 보호 |
| **인증** | 디바이스 인증, 자격증명 보호 |
| **DRM·무결성** | 플랫폼 신뢰 검증(Windows 11 요구) |

## 4. 관련 개념 비교

| 구분 | TPM | HSM |
|---|---|---|
| **형태** | PC/서버 내장 칩 | 전용 어플라이언스 |
| **용도** | 플랫폼 신뢰·무결성 | 대규모 키 관리·서명 |

## 5. 고려사항 및 시사점
- **하드웨어 신뢰 루트**로 제로트러스트 디바이스 인증 기반
- 펌웨어 취약점·물리 공격 대비, 키 백업·복구 정책
- 보안 부팅·기밀 컴퓨팅(Confidential Computing)의 토대

---

> **한 줄 요약**: TPM은 *키 보관·무결성 측정(PCR)·봉인·원격검증* 을 수행하는 하드웨어 보안 칩으로, 보안 부팅·디스크 암호화·디바이스 인증의 신뢰 루트(Root of Trust)를 제공한다.

# 트리 정렬(Tree Sort)

## 1. 개요

### 가. 정의
> 데이터를 **이진 탐색 트리(BST)에 삽입한 뒤, 중위 순회(In-order Traversal)로 읽어내어 정렬**하는 알고리즘. 이진 탐색 트리의 '왼쪽<루트<오른쪽' 성질을 정렬에 활용한다.

트리 정렬의 핵심 원리는 '**이진 탐색 트리를 중위 순회하면 자동으로 정렬된 순서가 나온다**'는 성질에 있다. 이진 탐색 트리는 어떤 노드든 왼쪽 서브트리에는 자기보다 작은 값, 오른쪽 서브트리에는 큰 값이 오도록 구성된다. 따라서 '왼쪽 → 루트 → 오른쪽' 순서로 방문하는 중위 순회를 하면, 가장 작은 값부터 큰 값까지 오름차순으로 읽힌다. 즉 정렬이라는 목적을 위해 별도의 비교·교환 로직을 짜는 대신, 데이터를 트리에 넣는 것만으로 정렬 구조가 만들어지고, 순회만 하면 정렬 결과가 나오는 것이다. 삽입할 때 자연스럽게 크기 순으로 자리를 찾아가는 자료구조의 성질을 정렬에 그대로 빌려 쓰는 우아한 방법이다.

### 나. 필요성
데이터가 계속 들어오는 상황에서 삽입과 정렬을 함께 관리해야 할 때, 트리 정렬은 삽입만으로 정렬 구조를 유지해 유용하다. 다만 성능은 트리의 균형에 크게 좌우된다.

## 2. 동작 원리


동작은 두 단계다. 먼저 데이터를 하나씩 이진 탐색 트리에 삽입한다(각 값이 크기 비교로 제자리를 찾아감). 그다음 트리를 중위 순회하면 오름차순으로 값이 출력된다. 예를 들어 5,3,8,1을 삽입하면 5를 루트로 왼쪽에 3(그 왼쪽에 1), 오른쪽에 8이 배치되고, 중위 순회하면 1,3,5,8이 나온다.

## 3. 복잡도

| 구분 | 시간 복잡도 | 비고 |
|---|---|---|
| **평균(균형 트리)** | O(n log n) | 삽입 각 O(log n) × n + 순회 O(n) |
| **최악(편향 트리)** | O(n²) | 이미 정렬된 입력 → 한쪽으로 치우침 |
| **공간** | O(n) | 트리 저장 |

트리 정렬의 성능은 **트리가 얼마나 균형 잡혔는지** 에 달려 있다. 무작위 데이터는 대체로 균형 잡혀 O(n log n)이지만, 이미 정렬된 데이터를 넣으면 트리가 한쪽으로 길게 치우친(편향) 연결 리스트처럼 되어 O(n²)로 악화된다.

## 4. 고려사항 및 시사점

1. **균형 트리로 최악을 회피**한다. AVL 트리·레드-블랙 트리 같은 자가 균형 이진 탐색 트리를 쓰면, 편향을 막아 항상 O(n log n)을 보장할 수 있다.
2. **다른 정렬과의 비교**를 이해한다. 힙 정렬도 트리 구조(힙)를 쓰지만 제자리 정렬이고, 트리 정렬은 추가 트리 공간이 필요하다. 무작위 데이터의 평균 성능은 퀵·병합 정렬과 유사하다.
3. **동적 데이터에 유리**하다. 데이터가 실시간으로 삽입·삭제되면서 정렬 상태를 유지해야 하는 경우, 균형 이진 탐색 트리 기반 트리 정렬이 삽입만으로 정렬을 유지해 효과적이다.

---

> **한 줄 요약**: 트리 정렬은 *이진 탐색 트리에 삽입 후 중위 순회로 정렬* 하는 알고리즘으로, 평균 O(n log n)이나 편향 시 O(n²)이므로 AVL·레드블랙 트리 같은 균형 트리로 최악을 회피하며 동적 데이터 정렬에 유리하다.

# 가상화(Virtualization)

## 1. 개요

### 가. 정의
> 물리적 자원(CPU·메모리·OS·애플리케이션)을 **논리적으로 추상화하여 하나를 여럿처럼, 또는 여럿을 하나처럼 사용**하게 하는 기술. 자원 활용률을 높이고 유연한 운영을 가능케 한다.

가상화의 본질은 '**물리적 실체와 논리적 사용을 분리하는 것**'이다. 한 대의 물리 서버 위에 여러 개의 가상 서버(VM)를 올리면, 놀고 있던 자원을 알뜰히 나눠 쓸 수 있고, 필요에 따라 자원을 즉시 늘리거나 옮길 수 있다. 이 추상화는 서버뿐 아니라 애플리케이션·데스크톱·네트워크·스토리지 등 여러 계층에서 이뤄진다. 특히 응용 프로그램 가상화는 프로그램을 OS에 직접 설치하지 않고 격리된 실행 환경에서 돌려, 설치 충돌 없이 어디서나 실행되게 한다. 가상화는 클라우드 컴퓨팅의 근간 기술로, "필요한 만큼 나눠 쓰고 유연하게 이동한다"는 클라우드의 탄력성이 모두 가상화에서 출발한다.

### 나. 일반적인 운영체제의 프로그램 동작 방식
일반 OS에서 프로그램은 OS가 관리하는 자원(CPU·메모리·파일시스템·레지스트리)에 직접 설치·연결되어 동작한다. 이 방식은 프로그램이 OS 환경에 강하게 묶여, 설치 시 다른 프로그램과 라이브러리·설정이 충돌하거나(DLL 지옥) 환경이 다르면 실행되지 않는 문제를 낳는다.

## 2. 응용 프로그램 가상화 동작 방식


응용 프로그램 가상화는 프로그램과 OS 사이에 **가상화 계층(격리된 실행 환경)** 을 두어, 프로그램이 OS에 직접 설치되지 않고 이 계층 안에서 실행되게 한다. 프로그램이 요구하는 파일·레지스트리·설정을 가상 계층이 대신 제공하므로, 실제 OS는 변경되지 않는다. 그 결과 설치 충돌이 없고, 하나의 패키지를 여러 PC에서 설치 없이 실행하며, 프로그램을 삭제해도 OS에 흔적이 남지 않는다. 스트리밍 방식으로 필요한 부분만 내려받아 실행하기도 한다.

| 구분 | 일반 방식 | 응용 프로그램 가상화 |
|---|---|---|
| **설치** | OS에 직접 설치 | 격리 환경에서 실행(무설치) |
| **충돌** | 라이브러리·설정 충돌 | 격리로 충돌 없음 |
| **이식성** | 환경 종속 | 어디서나 실행 |

## 3. 원격 데스크톱 프로토콜의 종류

데스크톱 가상화(VDI)에서 중앙 서버의 화면을 사용자 단말로 전송하는 것이 원격 데스크톱 프로토콜이다.

| 프로토콜 | 특징 |
|---|---|
| **RDP** | 마이크로소프트 표준, 윈도우 환경에 널리 사용 |
| **PCoIP** | 픽셀 단위 전송, 고화질·그래픽 작업에 강함 |
| **HDX / ICA** | 시트릭스 기반, 대역폭 최적화 |
| **SPICE** | 오픈소스(KVM), 리눅스 가상화 |

## 4. 가상화 유형

| 유형 | 내용 |
|---|---|
| **서버 가상화** | 하이퍼바이저로 물리 서버에 다수 VM |
| **데스크톱 가상화(VDI)** | 중앙 서버에서 데스크톱 제공 |
| **애플리케이션 가상화** | 프로그램을 격리 실행 |
| **컨테이너** | OS 커널 공유, 경량 격리(Docker) |

## 5. 고려사항 및 시사점

1. **VM과 컨테이너의 선택**이 중요하다. VM은 하이퍼바이저로 OS까지 가상화해 격리가 강하지만 무겁고, 컨테이너는 OS 커널을 공유해 가볍고 빠르므로, 격리 수준과 밀도 요구에 맞게 선택한다.
2. **클라우드·DevOps의 기반**이다. 가상화가 자원의 유연한 분할·이동을 가능케 해 클라우드 탄력성과 컨테이너 기반 배포(쿠버네티스)를 뒷받침한다.
3. **보안·성능 오버헤드를 고려**한다. 가상화 계층은 관리 편의를 주지만 성능 오버헤드와 새로운 공격 표면(하이퍼바이저 탈출 등)을 만들므로, 격리·모니터링을 강화해야 한다.

---

> **한 줄 요약**: 가상화는 *물리 자원을 논리적으로 추상화* 해 자원 활용과 유연성을 높이는 기술로, 응용 프로그램 가상화는 격리 환경에서 무설치·무충돌 실행을 제공하고 RDP·PCoIP·HDX 등 원격 프로토콜과 함께 클라우드·컨테이너의 기반이 된다.

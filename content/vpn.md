# VPN(Virtual Private Network)

## 1. 개요

### 가. 개념·특징
> 공용 네트워크(인터넷) 위에 **암호화된 가상의 전용 통신로(터널)** 를 구성해 안전하게 데이터를 주고받는 기술.

- **특징**: 터널링, 기밀성·무결성·인증, 전용선 대비 **저비용**

## 2. IPSec VPN vs SSL VPN

| 구분 | IPSec VPN | SSL VPN |
|---|---|---|
| **계층** | 네트워크(L3) | 전송/응용(L4~7) |
| **접속** | 전용 클라이언트 | 웹 브라우저(무설치) |
| **용도** | 지사간 상시 연결(Site-to-Site) | 원격 사용자 접속(Remote) |
| **범위** | 네트워크 전체 | 특정 애플리케이션 |

## 3. VPN 기술 요소

```mermaid
flowchart LR
  T[터널링<br/>L2TP·IPSec] --> E[암호화<br/>AES]
  E --> A[인증<br/>IKE·인증서]
  A --> I[무결성<br/>HMAC]
```

| 요소 | 설명 |
|---|---|
| **터널링** | L2TP·PPTP·IPSec(ESP/AH)·SSL/TLS |
| **암호화** | AES 등 대칭키로 기밀성 |
| **인증** | IKE·전자서명·인증서로 상호 인증 |
| **무결성** | HMAC·해시로 위변조 탐지 |
| **키관리** | IKE(Internet Key Exchange) |

## 4. 시사점
- 재택·원격근무 확산으로 **SSL VPN·ZTNA(제로트러스트)** 로 진화

---

> **한 줄 요약**: VPN은 *공용망 위에 암호화 터널로 가상 전용망을 구성* 하는 기술로, 네트워크 계층의 IPSec(지사간)과 응용 계층의 SSL(원격접속)이 있으며 터널링·암호화·인증·무결성을 핵심 요소로 한다.
